- CRITICAL — `scripts/compound-v-validate-manifest.py:810-853`, `schemas/fastpath-review-receipt.schema.json:5-19,30-68` — post-review validation does not enforce the receipt’s anti-replay bindings. It never compares `manifest_digest`, `baseline_sha`, or `final_diff_digest` with current artifacts, does not require `verdict == approved`, and verifies `digest` only when optionally present. A stale receipt—or one with `verdict:"issues"`—can pass. Fix: require the self-digest and approved verdict, recompute the manifest and final baseline-relative diff digests immediately before merge, and compare the baseline with immutable state.

- CRITICAL — `scripts/compound-v-fastpath-run.py:437-442,517-567` — malformed gate results fail open. Any dictionary lacking `escalate`, including `{}` or `{"error":"F2 crashed"}`, is treated as a clean F2 result. Likewise, only `status=="timeout"` blocks review; a result with `status:"error"` and `verdict:"approved"` returns `merge_ok:true`—confirmed directly. Fix: require `f2_result["escalate"] is False`, exact result schemas, `kind=="review_result"`, and `status=="ok"`.

- CRITICAL — `scripts/compound-v-taxonomy.py:369-412`, `scripts/compound-v-preeval.py:259-264,616-629`, `scripts/compound-v-postdiff-reclassify.py:556-564` — malformed taxonomies are normalized and used without running the structural taxonomy validator. `_has_safety_coverage` only checks that the sensitive list is nonempty. - HIGH — `scripts/compound-v-fastpath-materialize.py:251-340`, `scripts/compound-v-validate-manifest.py:982-1007` — the claimed tamper guard never verifies the pre-eval record’s self-digest, schema, low/low axes, `override_fired`, fan-out, or forbidden localization flags. - HIGH — `scripts/compound-v-preeval.py:550-568,682-710` — taxonomy pinning breaks across `needs_t3` resume. - HIGH — `scripts/compound-v-preeval.py:528-547,723-740` — Phase-P crash recovery is not idempotent. The record is written with `O_EXCL` before `predicted` is appended; a crash between those operations causes re-entry to rediscover the same ID and then fail on the existing record instead of appending the missing event. Completed identical requests are also rediscovered forever rather than receiving a fresh ID. Fix: reconcile each artifact/event independently, verify existing contents, append a missing predicted event idempotently, and ignore completed intents when minting a new request instance.

- HIGH — `scripts/compound-v-preeval.py:665-674`, `skills/compound-v/phase-preeval.md:30-36` — `pre_eval.enabled`, `pre_eval.fast_path`, and `pre_eval.remember` are loaded but never enforced by executable code; only `fan_out_threshold` and `token_cap` are read. - HIGH — `scripts/compound-v-triage-outcomes.py:331-360` — cohort separation confuses “eligible prediction” with “fast path accepted.” An eligible offer that the user declines and later binds to a normal full-pipeline run is classified as a fast-path parent because the bind carries no run kind.
