# Phase 1A — Code Archaeology: v2.9 Pre-Evaluation

Binding integration constraints from the real code (cite file:line in the build). Full audit ran
2026-07-11; this is the durable design-constraint extract every implementer MUST honor.

## Confirmed favorable
- **Phase enum is prose-only** — no script hard-codes `SPEC_READY…MERGED`. Adding
  `PRE_EVAL_DONE`(record-status)/`FASTPATH_DISPATCHED`/`ESCALATION_REQUIRED` is doc + reader work, not
  a schema migration. `scripts/compound-v-liveness.py` keys on per-job `status`, not run `phase`.
- **Validator `audits` check is LAX** — `compound-v-validate-manifest.py` only checks `audits` is a
  non-empty dict (no sub-key/path checks). A sentinel skip-record already passes; enforcing its SHAPE
  is net-new validation to ADD, in **block YAML** (the `_mini_yaml` fallback mis-parses flow `{}`).
- Reuse primitives, don't recopy: `_seg_is_literal`/`glob_match` (validate-manifest.py),
  `matches`/`is_allowed` (scope-check.py), `load_config_models` (resolve-model.py), `append_line`
  (update-memory.py:112 — "ONLY appends, NEVER rewrites"), `compound-v-run-with-timeout.py`.

## Binding hazards (design constraints)
- **H1/AC-4:** `manifest.yaml` is read-only after materialization (`execution-manifest.md:61`).
  Escalation MUST mint a new run-id + link `escalated_to`, never mutate in place.
- **H2/AC-2:** the run dir / run-id does NOT exist pre-brainstorm (created post-plan). `pre-eval`
  records live under `docs/superpowers/pre-eval/`, keyed by a write-once `pre_eval_id`.
- **H7/AC-5:** the post-hoc reclassifier is a SIBLING analyzer, never an extension of
  `scope-check.py` (a hardened name-only gate with reproduced-exploit selftests). Reuse its changed-
  path SET; get size from a separate `git diff --numstat`; structural is net-new.
- **H8/AC-6:** PRE_EVAL is description-driven + UNENFORCEABLE (only the reminder-only
  `brainstorm-trigger0-nudge.sh` PreToolUse hook exists). Safe ONLY via fail-closed → FULL_PIPELINE.
  Never claim enforcement.
- **H9/AC-3:** all jsonl is strictly append-only. `triage-outcomes` is a THREE-event protocol
  (`predicted`→`bind`→`actual`) joined on `pre_eval_id`, never a mutated line.
- Two mandatory commit points (v2.6.4) are load-bearing; every new record is written AND committed
  (two commands, no `&&`, each exit code checked) or `git worktree remove` deletes it.
- `state-machine.md`, `commands/v-status.md`, `commands/v-resume.md`, `agents/parallel-dispatcher.md`
  MUST all learn the new states/transitions or status/resume behave undefined.
