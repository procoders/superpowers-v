## 1. Goal

Ship **v1.0.0**: evolve Compound V from description-driven prose into a lightweight, manifest-driven execution orchestrator — without a daemon, MCP server, or fabricated metrics (anti-ruflo charter). The three pre-flights and `/v:archaeology` stay behaviourally unchanged; the orchestrator extends the tail (manifest → dispatch → scope-gate → collect → review → memory) with multi-backend execution (Claude + Codex), per-job isolation, crash-resume, and `/v:init`.

## 2. Pre-flight audit — binding design constraints

From the three audits run against this repo. These are non-negotiable for every task.

**Archaeology (existing-code reality):**
- MUST keep the three pre-flights (1A/1B/1C agents + parallel dispatch + output dirs) and `/v:archaeology` **behaviourally unchanged**.
- MUST preserve **description-based auto-fire**: `SKILL.md`'s description still triggers on brainstorming/writing-plans/about-to-execute, and **every existing `evals.json` case must still pass**.
- MUST evolve `/v:dispatch` to be manifest-aware **backward-compatibly** — a bare plan path still works (the `plan-saved-nudge` hook and current users pass plan paths).
- MUST bump **both** `plugin.json` and `marketplace.json` to `1.0.0` in lockstep; add the `orchestrator` keyword.
- MUST keep every new intra-plugin `.md` link resolvable, every `hooks/*.sh` executable + shellcheck-clean.
- MUST NOT specify **Haiku** in any frontmatter (both `lint-frontmatter.py` and `validate.yml` reject it).

**Domain (plugin-authoring reality):**
- A **sub-skill is a sibling directory** (`skills/backend-launcher/`), referenced by prose "read file X and apply" — there is no skill-import API.
- The **scope gate is a deterministic script** the dispatcher calls after every job: `git diff --name-only HEAD` ∪ `git ls-files --others --exclude-standard` vs `write_allowed`. Prose SCOPE LOCK is advisory; the script is the authority. BLOCKED halts, never merges.
- Codex is a **Bash-spawned `codex exec` worker** (own process, own worktree) — never an `agents/` entry, never the openai-codex broker.
- Reviewers are **always `model: opus`**; manifest `backend`/`model` values (`gpt-5.5`, etc.) **never appear in frontmatter** — they are execution-layer data.
- Hooks stay at SessionStart + PostToolUse(Write), each preserving the **3-shape platform JSON** (Cursor/Claude/generic). No new hook events, no fabricated hook env vars.
- Config is split: project `.claude/compound-v.json` (stance, committed) + user-level `~/.claude/compound-v-capabilities.json` (capability cache, uncommitted).

**Toolchain (verified live on this machine — see §3):** codex-cli 0.130, git 2.50.1, **bash 3.2.57** (stock macOS — no bash-4 features), **python3 3.9.6** (no `match`, no `X|Y` unions), jq 1.7.1, Context7 MCP available.

## 3. Verified toolchain facts (pin these — checked live, do not re-derive)

| Fact | Detail | Consequence |
|---|---|---|
| **`codex exec` rejects `--ask-for-approval never`** | It's a top-level/interactive flag, absent from `codex exec --help`; `exec` already defaults to `approval: never`. The PRD's original adapter would fail on **every** Codex job. | **Drop the flag.** If a non-default policy is ever needed: `-c approval_policy=never`. |
| **Verified working `codex exec` flag set (0.130)** | `--cd <wt> --sandbox workspace-write --skip-git-repo-check --model <m> [--output-schema <f>] --output-last-message <f> -c sandbox_workspace_write.network_access=<bool> "$prompt"` | Pin exactly; `/v:init` re-probes against the **EXEC subcommand** help (the top-level merge is what masked the bug). |
| **Resume** | `codex exec resume <SESSION_ID> [PROMPT]` (UUID printed in the run banner) or `--last`. **No `--session-id` flag.** | Capture session_id from the banner / output. |
| **`git worktree diff` does not exist** | Use plain `git -C <wt> diff --name-only` + `git -C <wt> ls-files --others --exclude-standard`. | Scope-gate + merge use plain git inside the worktree. |
| **Stock macOS = bash 3.2.57 / python 3.9.6** | No associative arrays, `mapfile`, `${var,,}`; no `match`/PEP-604 unions. | All helper scripts target bash 3.2 + py 3.9, stdlib only (+ optional pyyaml with fallback). |
| **codex emits a cosmetic `[features].codex_hooks is deprecated` stderr warning** | Pollutes captured output if not handled. | Worker script suppresses/ignores it. |
| **Context7 MCP namespace** | `plugin_context7_context7` / `plugin:context7:context7` (plugin-namespaced) | `/v:init` detection matches the namespace, not bare `context7`. |
| **Workflow tool not exposed in a plain subagent shell** | Confirms Engine C is capability-gated. | Accelerator is opt-in, capability-probed, **always** falls back to Engine A. |
| **`--output-schema` works** | Accepts strict JSON Schema (`additionalProperties:false` + `required`). | Author `job_result.schema.json` strict — but see resolved Q below: schema drives only the human summary, not enforcement. |

## 4. Resolved open questions (PRD §11 + audit follow-ups)

1. **Naming** → keep `backend-launcher` (literal sub-skill name = directory = invocation token = cross-ref path).
2. **Config** → project `.claude/compound-v.json` (stance) + user `~/.claude/compound-v-capabilities.json` (capability cache).
3. **Worktree merge-back** → on PASS: `git -C <wt> diff HEAD | git apply` into the main tree, then `git worktree remove -f`. On BLOCKED: leave the worktree for inspection, do not merge. (Loses per-job commit attribution — acceptable for disjoint file sets.)
4. **Worktree storage** → a temp dir **outside** the repo (`$TMPDIR/compound-v/<run-id>/<job-id>`); no `.gitignore` change needed.
5. **Resume tie-break** → **git-wins.** If `state.json` says `done` but the files aren't in git, re-dispatch (resume stays idempotent).
6. **Codex `job_result` enforcement** → `files_changed` / `violations` / `blocked` are **git-derived** (authoritative); the model's `--output-last-message` text feeds only the human `summary`. Never trust the model to self-report what it changed.
7. **Workflows probe** → default **off**; attempt-and-catch behind a capability probe, fall back to Engine A on any failure.

## 7. Cross-cutting requirements (every task)

- **No Haiku** in any frontmatter. Reviewers/agents = `model: opus`. Manifest `backend`/`model` are execution data, never frontmatter.
- **Scripts target bash 3.2 + python 3.9, stdlib only** (no npm/pip mandatory deps; pyyaml optional with fallback). Suppress codex's deprecation stderr.
- **Scope gate runs on every job**; BLOCKED halts the run. Enforcement fields are git-derived, never model-self-reported.
- **Anti-ruflo:** no daemon, no MCP server, no vector DB, **no printed/fabricated token-cost metrics** (CI grep gate enforces this on scripts + docs).
- **Dead-link CI** runs at the final integration pass, not per-task (gap-fix G2); within a batch, tasks link only to earlier-batch / Task-0 outputs.
- **Backward compatibility:** `/v:dispatch`, the evolved agents, and the hooks must still accept the 0.1.x plan-path flow.

## 9. Risks & regression surface (from archaeology)

- Rewriting `SKILL.md`'s description for orchestrator-as-default could degrade the auto-fire triggers → **evals.json is the guard; run it.**
- Evolving the 3 agents changes their inputs (plan → manifest) → **must accept both** or external callers + the hook break.
- Introducing worktrees changes the merge model (0.1.x did direct writes only) → the `git apply`-into-main default (Q3) plus per-job isolation choice is the mitigation; direct Claude jobs and worktree jobs must not collide at merge.
- Hook edits risk the 3-shape jq branches / shellcheck → additive changes only.
- New `execution/` + `memory/` dirs must not contradict `SKILL.md`'s Output Directory Conventions → updated in Task 0.
