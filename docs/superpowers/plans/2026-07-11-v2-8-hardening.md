## Shared Interface Contract (exact strings all tasks rely on)

- **Recon doc sections (now genuinely five, closing A20/E-C1-16):** the anti-anchoring header, then verbatim `## QUESTIONS TO ASK`, `## VERIFIED FACTS / CONSTRAINTS`, `## UNVERIFIED LEADS`, `## SUGGESTED DIRECTIONS`, `## SOURCES`.
- **Epistemic rule:** VERIFIED = claim checked against a cited primary source (provisionally binding; 1B/1C revalidate). UNVERIFIED LEADS = everything else; leads must become questions until validated.
- **Fail-closed config rule (verbatim, used by Tasks 1/2/7):** "Missing file or key → the documented defaults (`deep_research: \"ask\"`, `batch_elicitation: true`). Malformed JSON, wrong type, or unknown value → warn once, then use `deep_research=ask` and `batch_elicitation=false` for this session; never treat an invalid value as `auto`."
- **Recon path handoff:** the caller stores the exact recon path in the brainstorm's working state and the spec's metadata; scanning `docs/superpowers/recon/` is fallback-only.
- **recon-outcomes stream:** `docs/superpowers/memory/recon-outcomes.jsonl`, one JSON line `{ts, topic, outcome, engine, path, consumed}` with `outcome ∈ fired|plumbing_skip|kb_skip|off|declined|no_engine|saved`; never read by routing.
- **Effort vocabulary:** `xhigh` is valid **iff** `backend: codex`; every other backend rejects with a clear error naming the rule.
- **Hook file:** `hooks/brainstorm-trigger0-nudge.sh`, registered in `hooks/hooks.json`; injects a one-line idempotent reminder ("run the Trigger 0 gates from phase-0-recon.md if not already done") when the Skill tool invokes `superpowers:brainstorming`.
