# v2.9 Pre-Evaluation & Proportionate Fast-Path ŌĆö Implementation Plan (rev 3)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a fast, cheap Pre-Evaluation stage that scores each request on two axes and OFFERS a
proportionate fast-path only when a change is provably trivial + low-impact, with a real escalation
safety net and onboarding-manufactured signals.

**Architecture:** A pre-brainstorm scoring engine + bounded localization writing a **write-once**
`pre-eval/<pre_eval_id>.json` (keyed independent of run_id); a deterministic truth-table (no raw LLM
magnitude); three prose-only state-machine states with escalation that **mints a new run-id** (never
mutates the frozen manifest); a **conditional** fast-path manifest (relaxed spec/plan fields, review
modeled as a dispatcher phase outside `jobs`); a sibling post-hoc diff re-classifier sequenced
**before** merge/worktree-removal against the pinned baseline; append-only two-event triage-outcomes
with cohort-separated Tier 2.

**Tech Stack:** Python 3.9-safe stdlib (soft-PyYAML fallback), YAML taxonomy, append-only JSONL,
the existing manifest/state-machine/scope-gate substrate, FTS5 V-memory, one `light`-tier (Sonnet)
classify ŌĆö invoked by the **parent harness as a Task**, not from Python.

> **rev 2** folds in Codex plan-review round 1 (reject, 3 crit + 9 high ŌĆö all accepted), marked
> `[CR1-n]`. **rev 3** folds in round 2 (reject, "closes most round-one objections", 3 crit + 6 high
> + 2 med ŌĆö all accepted), marked `[CR2-n]`: CR2-1 DAG C2 edge + own wave; CR2-2 Task M1 materializer;
> CR2-3 validator cross-artifact binding (AC-13); CR2-4 idempotent two-phase escalation (AC-15);
> CR2-5 review as `needs_review` Task (not Python); CR2-6 immutable taxonomy snapshot+digest; CR2-7
> bounded regex (safe subset + killable subprocess, AC-16); CR2-8 `PRE_EVAL_DONE` as record status;
> CR2-9 typed analyzer = Python-`ast` real, others fail-closed; CR2-11 shared `load_project_config`.
> The spec was amended in lockstep (AC-3 three-event telemetry; AC-7/13/14/15/16).

## Global Constraints

- Model policy: Opus default; T3 classify = `light`/Sonnet via an in-harness Task; **never Haiku**.
  Reviewers always `deep`/opus (manifest invariant #4).
- No fabricated metrics: the 1-10 is a band-midpoint label; `/v:status` precision uses git-derived
  actuals only, "insufficient samples" below the floor.
- External-launch invariant: any external CLI (codex T3 alternative, churn git subprocess) routes
  through `scripts/compound-v-run-with-timeout.py`, `stdin </dev/null`.
- Git-derived enforcement, never model self-report.
- Commit-discipline (v2.6.4): every new record written AND committed via two separate commands
  (no `&&`), each exit code checked.
- Fail-closed everywhere: any ambiguity/missing-data/tier-disagreement/`unknown`/budget-overrun/
  unsupported-language/parse-failure ŌåÆ `FULL_PIPELINE` (or escalate, post-diff). Never fail open.
- Reuse (no new copies): `_seg_is_literal`/`glob_match` (`validate-manifest.py`), `matches`/
  `is_allowed` (`scope-check.py`), `load_config_models` (`resolve-model.py`), `append_line`
  (`update-memory.py`), `compound-v-run-with-timeout.py`.
- Python 3.9-safe, stdlib-first; PyYAML optional (soft-import + stdlib fallback), never hard
  `import yaml`.
- Spec: `docs/superpowers/specs/2026-07-11-v2.9-pre-evaluation-design.md` (┬¦0 corrections binding;
  AC-1ŌĆ”AC-12 are the acceptance gates).
