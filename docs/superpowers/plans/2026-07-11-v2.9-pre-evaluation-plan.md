# v2.9 Pre-Evaluation & Proportionate Fast-Path — Implementation Plan (rev 5)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a fast, cheap Pre-Evaluation stage that scores each request on two axes and OFFERS a
proportionate fast-path only when a change is provably trivial + low-impact, with a real escalation
safety net and onboarding-manufactured signals.

**Architecture:** A pre-brainstorm scoring engine + bounded localization writing a **write-once**
`pre-eval/<pre_eval_id>.json` (keyed independent of run_id); a deterministic truth-table (no raw LLM
magnitude); three prose-only state-machine states with escalation that **mints a new run-id** (never
mutates the frozen manifest); a **conditional** fast-path manifest (relaxed spec/plan fields, review
modeled as a dispatcher phase outside `jobs`); a sibling post-hoc diff re-classifier sequenced
**before** merge/worktree-removal against the pinned baseline; append-only two-event triage-outcomes
with cohort-separated Tier 2.

**Tech Stack:** Python 3.9-safe stdlib (soft-PyYAML fallback), YAML taxonomy, append-only JSONL,
the existing manifest/state-machine/scope-gate substrate, FTS5 V-memory, one `light`-tier (Sonnet)
classify — invoked by the **parent harness as a Task**, not from Python.

## Global Constraints

- Model policy: Opus default; T3 classify = `light`/Sonnet via an in-harness Task; **never Haiku**.
  Reviewers always `deep`/opus (manifest invariant #4).
- No fabricated metrics: the 1-10 is a band-midpoint label; `/v:status` precision uses git-derived
  actuals only, "insufficient samples" below the floor.
- External-launch invariant: any external CLI (codex T3 alternative, churn git subprocess) routes
  through `scripts/compound-v-run-with-timeout.py`, `stdin </dev/null`.

  (no `&&`), each exit code checked.
- Fail-closed everywhere: any ambiguity/missing-data/tier-disagreement/`unknown`/budget-overrun/
  unsupported-language/parse-failure → `FULL_PIPELINE` (or escalate, post-diff). Never fail open.

  `is_allowed` (`scope-check.py`), `load_config_models` (`resolve-model.py`), `append_line`
  (`update-memory.py`), `compound-v-run-with-timeout.py`.
- Python 3.9-safe, stdlib-first; PyYAML optional (soft-import + stdlib fallback), never hard
  `import yaml`.
- Spec: `docs/superpowers/specs/2026-07-11-v2.9-pre-evaluation-design.md` (§0 corrections binding;

## Lifecycle & commit-ordering protocol (SINGLE AUTHORITY — rev 5, resolves the CR4 ordering cluster)

Every task below defers to THIS ordering; where a task's prose disagreed, this section wins. Owners
in brackets. All commits use the two-command discipline; every boundary is a resume/crash checkpoint.

**Phase P — pre-run (parent-owned; NO run_id yet; all artifacts under `docs/superpowers/pre-eval/`):**
1. `localize` → write+commit `pre-eval/<pre_eval_id>.localization.json` (write-once). [A1]
2. **Content-address the taxonomy HERE, pre-run** [CR4-2]: copy the current taxonomy to
   `pre-eval/<pre_eval_id>.taxonomy-snapshot.yaml`, compute `taxonomy_digest`, write+commit. This
   immutable snapshot lives in the pre-eval area (NOT under a run — a declined request never makes a
   run). [A1/A3]
3. `score` → write+commit write-once `pre-eval/<pre_eval_id>.json` (`status:PRE_EVAL_DONE`,
   `taxonomy_ref`→the snapshot, `taxonomy_digest`, `decision`). [A3]
4. append+commit `predicted` event keyed by `pre_eval_id`. [A3]
Resume in Phase P: discover partial state by `pre_eval_id`, continue from the first missing artifact.

**Phase M — materialize (ONLY on accepted `FASTPATH_ELIGIBLE`):** [M1]
5. mint a **deterministic** run-id from `pre_eval_id`.
6. copy the pre-eval taxonomy snapshot into the run (preserving `taxonomy_ref`/digest); write stubs,
   block-YAML skip-records, the `fast_path` manifest carrying the review **DECLARATION only**
   (`fast_path.review:{backend:claude, tier:deep}` OR `model:opus` — **no receipt yet**), captured
   prompt.
7. commit ALL run artifacts **except** `state.json`.
8. append+commit `bind` event `{pre_eval_id, run_id}` — **BEFORE** state [CR4-3].
9. commit `state.json` at `FASTPATH_DISPATCHED` **LAST**. Invariant: *committed `state.json` ⇒ `bind`
   already durable ⇒ run complete.* Resume: a run dir without a committed `state.json` is incomplete →
   rebuild deterministically (same `pre_eval_id`→same run-id, discover existing child before minting).

**Two validation modes** [CR4-1]: [C1]
- **pre-dispatch:** validate the review DECLARATION (`backend:claude`+`tier:deep` OR `model:opus`
  [CR4-8]) + all cross-artifact bindings + containment; **forbid/ignore any receipt** (it can't exist
  yet).
- **post-review:** require + verify the dispatcher-generated **invocation receipt** naming the
  resolved model **before** `REVIEWED`/`MERGED`.

**Dispatch sequence — the ONE authoritative order** [CR4-9]: [C2/H1/Z1 all match this]
`implementer → tests (floor) → scope gate → F2 (pinned baseline, pre-merge) → review (needs_review
Task; writes receipt) → post-review validation (receipt) → final scope recheck → merge → append+commit
terminal `actual`` [CR5-4: the terminal `actual` is emitted ONLY AFTER the merge/commit boundary
succeeds — a pre-merge `reviewed`/`merge_pending` event (which precision IGNORES) may be recorded
earlier, but a `review_passed` `actual` that never merged must never reach Tier 2]. On F2 escalate:
two-phase escalation (AC-15); append+commit `actual` with `escalated:true` on the PARENT.

**Event keying** [CR4-5]: `bind`/`actual` keyed by **`(pre_eval_id, run_id)`**. An escalation child
gets its OWN run-id; the parent's `actual` records `escalated:true`. **Precision is computed from the
fast-path PARENT outcome only**; the full-pipeline child contributes escalation evidence, never
low-corroboration.

**Path containment** [CR4-6]: every `*_ref` and the sole `write_allowed` literal MUST be normalized,
repo-relative (no absolute, no `..` segment), realpath-under-repo-root, a committed **regular file**
(not an escaping symlink) in its designated dir. Reuse scope-check's escaping-symlink scan; do not
rely on `_seg_is_literal` alone (it only rejects `*?[`).
