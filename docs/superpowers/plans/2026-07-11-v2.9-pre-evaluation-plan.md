# v2.9 Pre-Evaluation & Proportionate Fast-Path — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a fast, cheap Pre-Evaluation stage that scores each request on two axes and OFFERS a
proportionate fast-path only when a change is provably trivial + low-impact, with a real escalation
safety net and onboarding-manufactured signals.

**Architecture:** A new pre-brainstorm scoring engine + bounded localization writing a pre-run
`pre-eval/<slug>.json`; a deterministic truth-table (no raw LLM magnitude); three prose-only
state-machine states with escalation that mints a new run-id (never mutates the frozen manifest);
a fast-path skip-contract keeping the committed manifest + name-only scope gate + a proportionate
Opus review; onboarding drafts a taxonomy (path + content patterns) and a normalized churn cache;
a sibling post-hoc diff re-classifier; append-only two-event triage-outcomes for Tier 2.

**Tech Stack:** Python 3.9-safe stdlib (soft-PyYAML fallback), YAML taxonomy, append-only JSONL,
the existing manifest/state-machine/scope-gate substrate, the FTS5 V-memory index, one `light`-tier
(Sonnet) classify call routed through the timeout supervisor.

## Global Constraints

- Model policy: Opus default; T3 classify = `light`/Sonnet; **never Haiku** (CI-enforced). Reviewers
  always `deep`/opus (manifest invariant #4).
- No fabricated metrics (anti-ruflo): the 1-10 is a band-midpoint label; `/v:status` precision uses
  git-derived actuals only, "insufficient samples" below the floor.
- External-launch invariant: every external CLI (T3 codex classify, churn git subprocess) routes
  through `scripts/compound-v-run-with-timeout.py` with `stdin </dev/null`.
- Git-derived enforcement, never model self-report (scope gate + post-hoc reclassifier read git).
- Commit-discipline (v2.6.4): every new record (`pre-eval/*.json`, `triage-outcomes.jsonl`,
  `churn-cache.json`, taxonomy, ADRs) is written AND committed via two separate commands (no `&&`),
  each exit code checked — uncommitted artifacts are deleted by `git worktree remove`.
- Fail-closed everywhere: any ambiguity/missing-data/tier-disagreement/`unknown`/budget-overrun →
  `FULL_PIPELINE`. Never fail open.
- Reuse (no new copies): `_seg_is_literal`/`glob_match` (`validate-manifest.py`), `matches`/
  `is_allowed` (`scope-check.py`), `load_config_models` (`resolve-model.py`), `append_line`
  (`update-memory.py`), `compound-v-run-with-timeout.py`.
- Python 3.9-safe, stdlib-first; PyYAML optional (soft-import + stdlib fallback like
  `validate-manifest.py`), never a hard `import yaml`.
- Spec: `docs/superpowers/specs/2026-07-11-v2.9-pre-evaluation-design.md` (§0 corrections are
  binding; AC-1…AC-12 are the acceptance gates).
