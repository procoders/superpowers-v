## 🔴 Findings + RESOLVED decisions

- **🔴-1 "same label ≠ same fact".** `_aggregate` keys on family+disposition string only (968-971), never
  inspects `evidence` (captured per ballot at 744/764). Two correlated families can both emit
  `blocked_external` on *different hallucinated* missing facts → false confirm. **RESOLVED (user Q2):
  closed `blocker_category` enum + same-category agreement** — `credential | external-account | infra |
  third-party-data | legal-approval | human-decision`; category mismatch or vague/null → SUSPECTED →
  human. This is the highest-value hardening.
- **🔴-2 done_with_blockers as merge-routed success = reward-hack escape hatch.** `blocked_external` is the
  SOFTEST disposition (CONSERVATISM_RANK 1) yet v2.14 routes it to the MOST-complete outcome (merge). A
  gamed blocker is a PASS-shaped terminal no one audits, reached by a FAIL disposition.
  **RESOLVED (user Q1): auto-merge on ≥2-family confirm** — user chose max autonomy. **Guardrails KEPT
  (compatible with auto-merge):** (a) PASS-integrity sample-audit ALWAYS over-samples a `confirmed`-blocked
  feature (never in the "not sampled" bucket); (b) finishing-a-development-branch / final report SURFACES
  the blocked remainder (feature, category, families_agreeing, evidence) for human eyes — never silently
  dropped. `confirmed = ... and retry_n == 0` (any single retry_fix ballot blocks confirm) is retained.
- **🔴-3 headless shim: "normal posture" stalls (no TTY); naïve fix = the repo-deleting flag.** Confirmed:
  `claude -p` under a no-TTY parent hangs on permission prompts (#56540/#9026); `--dangerously-skip-permissions`
  is NOT even the headless flag (#52506 — still prompts on first use). **RESOLVED (user Q3): `--permission-mode
  dontAsk` + curated read-mostly `--allowedTools` allowlist** — actually runs, refuses everything off-list,
  never bypasses. Runbook carries a prominent DO-NOT block referencing the 2026-07-13 repo-deletion incident.
