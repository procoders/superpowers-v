## C2 — Design findings

1. **HIGH — Gate 2 has no freshness rule.** Topical coverage alone = strong hit; a stale
   recon on a fast-moving SDK suppresses live research indefinitely AND is handed to the
   brainstorm. Fix: strong hit = scope AND freshness; volatile material (libraries, APIs,
   regulations, availability, best practices) older than ~30 days degrades to partial —
   evidence still, skip-authority no longer.
2. **MEDIUM — Gate 1 skips exactly the plumbing most needing research** (tool migrations,
   bundler/test-runner choices are version-sensitive). Narrow: skip only trivial/already-
   specified plumbing; any tool choice/migration/version/compat/security question → gate 2.
3. **MEDIUM — `off` semantics:** gate 2 still injects recalled recon under `off` (state
   "off disables external recon; local recall still applies" — or reorder); invalid values
   (typo `"of"`) must fail CLOSED (treat as `off`/warn, never as `auto`).
4. **HIGH — Epistemic self-contradiction:** header says FACTS/CONSTRAINTS are binding;
   the same doc calls recon claims "unverified leads". Most dangerous anchor = a confidently
   mislabeled constraint. Fix: VERIFIED (primary-source, provisionally binding) vs
   PROVISIONAL/UNVERIFIED LEADS (must become questions until 1B/1C validate).
5. **HIGH — Anti-anchoring is procedural, not structural.** Same context synthesizes and
   then brainstorms; "non-exhaustive" doesn't remove an anchor; 2–3 directions may be three
   variations of one frame. Fix (accepted): **directions-late protocol** — brainstorm first
   from verified constraints + questions only; produce ≥3 first-principles proposals incl.
   one that deliberately rejects the recon framing; only then read SUGGESTED DIRECTIONS as
   a coverage/novelty check. State residual risk. (Fresh-context ideator REJECTED: over-eng.)
6. **MEDIUM — Psychological co-presence dependence:** answer-interaction test misses
   priming between visually co-present groups. Add second eligibility test: "could seeing
   one group's framing plausibly shift another answer?" → sequential.
7. **HIGH — No transactional answer protocol:** events carry bare `choice:"a"` with no
   group identity; partial answers undefined; per-screen clearing races a still-clicking
   user. Fix: globally unique group/choice ids (`data-choice="group:option"`), terminal
   submission = completion barrier, snapshot+reconcile before any push, never infer
   defaults for unanswered groups (ask sequentially / mark deferred).
8. **HIGH — Epic copy is factually wrong:** during upfront specs only sibling recon/spec
   docs accumulate — per-feature 1B/1C audits do NOT exist yet; thematically-close sibling
   recon is a false-positive strong-hit risk. Fix: sibling epic recon = partial by default;
   strong only if it covers the feature-specific delta (actor, workflow, external systems,
   constraints, currentness).
9. **Validation gap:** Trigger 0 shipped with zero end-to-end execution. Minimal live
   protocol: unfamiliar topic under `ask` (accept w/ narrowed scope) → engine runs bounded →
   contract-valid doc committed → brainstorm consumes questions-first → repeat related topic
   (KB-hit + freshness) → `off` negative control → one engine-failure path.

## C2 — Improvement proposals (arbitrated)

- ACCEPTED: freshness+feature-delta strong-hit rule; verified/provisional + directions-late;
  codex-only `xhigh` effort (all layers in lockstep); separate `recon-outcomes.jsonl`
  (`fired|plumbing_skip|kb_skip|off|declined|no_engine|saved` + engine + path + consumed) —
  NEVER fed into routing (scorecard stays implementation-only).
- REJECTED: `/v:recon` command (user call: plain-language ask + freshness rule + hook
  backstop cover the use cases; 14 commands already); adversarial CI fixtures (local suite
  suffices); fresh-context ideator (over-engineering).

## C1 — Executability findings (28; deduped against F1/C2 where convergent)

Trigger/eligibility: (1) "no grounding" (phase-0) vs "every feature topic" (SKILL) drift —
no grounding test exists; gates 1–3 ARE the complete list. (2) "skip silently" vs mandatory
log line vs Phase 0 announcement conflict — announce only when gates decide to RUN.
(3) plumbing boundary not operational — adopt "cannot alter shipped artifact/runtime/release
semantics/security/availability/perf" rule. (28) SKILL.md:250 "runs unchanged" contradicts
the deliberate elicitation override — "unchanged except the gated elicitation override".

Gate 2: (4) no executable strong-hit rule (open top results; same product/domain + same task
class + current framework constraints; rank alone never suffices). (5) memory-command failure
has no transition (missing/nonzero/invalid JSON → `KB=unavailable`, warn once, continue to
gate 3; never infer a hit from a failed lookup).

Gate 3 / consent: (6) malformed config undefined — missing = defaults; malformed/wrong-type/
unknown enum = warn + `deep_research=ask`(never auto) + `batch_elicitation=false` for the
session; `off` is a confidentiality control. (7) no-reply semantics — one blocking choice;
cancel/timeout/empty/unrelated-next-message = skip; narrowed-scope requires nonblank text
(ask once, then skip).

Engines: (8) "never delay the first question" contradicts recon's nature — bounded delay
with per-rung timeouts; cancel a timed-out rung. (9) A-fails→B fallback breaks the "one
engine, OR-bounds" budget — at most one engine COMPLETES; record both attempts; discard
incomplete A unless individually-sourced (label `partial`). (10) WebSearch denial/quota:
no retry; partial success → `PARTIAL RECON` with sourced claims only; report the real
reason, never "no engine available". (11) SKILL summary drifts from authority (`/v:remember`
vs direct command; 3–6 vs ≤6; "≤~150" vs exact 150) — make the summary literal.

Output: (12) slug/topic derivation absent (effective scope = narrowed if given; lowercase,
unicode-normalize, non-alnum→`-`, ≤60 chars, hash fallback; repo-local date). (13) create
dir; never overwrite; collision → unique suffix. (14) bare `git commit` can open an editor;
`&&` chaining violates the repo's own no-chained-side-effects rule — separate `git add -- p`
then `git commit -m … -- p`, check exit codes. (15) commit failure: announce "written but
not committed: <reason>", continue brainstorm, never claim committed/indexed. (16) five-vs-
four sections — say four, or add a fifth verbatim heading (`## UNVERIFIED LEADS` pairs with
the epistemics fix). (17) = C2#4. (18) claim→source mapping (`[F1]` ids + accessed date +
exact claim; "verified manually" names the artifact). (19) "≥2–3" ambiguous → "at least 3
materially divergent" (arbitrated: at least 2, prefer 3 — small topics can't always yield 3).
(20) later lookup uses undefined fuzzy matching — store the exact recon path in the
brainstorm's working state + spec metadata; scanning is fallback-only.

Elicitation: (21) no checkpoint algorithm — at each design checkpoint list candidate
questions, build a pairwise dependency graph, batch only isolated nodes, recompute after
each sequential answer. (22) condition-2 vs fallback contradiction — ARBITRATED: the
independence+count+config gate decides BATCHING; companion acceptance gates only the
companion SURFACE; fallback ladder chooses the surface (companion → AskUserQuestion →
sequential); SKILL/v-init updated to match. (23) >5 overflow — batch first ≤5 eligible at
one checkpoint; overflow one-at-a-time; no second batch from the same checkpoint.
(24) acceptance = explicit acceptance in THIS conversation AND a recorded `state_dir`;
unknown ⇒ companion unavailable. (25) namespaced ids + multiselect toggle replay (or
`selected` field). (26) reconciliation procedure — read+parse events BEFORE any push;
resolve per group; acknowledgement ≠ override; ambiguity → re-ask that group sequentially.
(27) companion runtime failures (server dead, write/read fails) → descend the ladder;
"AskUserQuestion" = the harness's structured-question equivalent, else terminal.

CLEAN (both passes): gate order agreement across surfaces; `off` egress semantics
consistent; live-listing authority; recon excluded from routing; answer-interaction rule +
tiebreak internally sound; `batch_elicitation=false` restores sequential everywhere;
terminal-over-form precedence; /v:init writes documented keys/defaults.
