# v2.14.0 — Confirmed Blockers (2nd external family) + Headless Resurrection Shim + Doc Refresh

**Goal:** Give Compound V's Epic Autonomous mode a genuinely *confirmed* blocker terminal
(`done_with_blockers`) backed by **two distinct external model families**, ship an **opt-in headless
`launchd`/cron shim** so a marathon can resurrect while the desktop app is closed, and **refresh the 5
stale architecture-doc targets** — all as one release, `2.14.0`.

**Architecture:** Three disjoint workstreams. **A** extends the existing v2.10 arbiter panel
(`compound-v-epic-arbiter.py`) to poll a **second external family** (Gemini via `agy`, read-only
advisory) alongside Codex (GPT), un-gates the *derived* `confirmed` path in `compound-v-epic-state.py`,
and adds the `done_with_blockers` terminal to the state spine + `/v:epic` driver. **B** adds a
**present-only generator** (`compound-v-headless-shim.py`) that prints a macOS `launchd` plist / Linux
cron entry + runbook — the user installs it; the plugin never `launchctl load`s anything and never emits
`--dangerously-skip-permissions`. **C** runs `/v:onboard --refresh` **last** (after A+B land) so the
regenerated citations reflect the final code.

**Tech Stack:** Python 3.9-safe stdlib only (no `match`, no `X | Y` unions); Bash (`bash -n` clean);
the existing `compound-v-run-with-timeout.py` supervisor; the existing capability-detection files
(`~/.claude/compound-v-capabilities.json`, `.claude/compound-v.json`). No new dependencies, no services.

## Global Constraints

- **Opus by default; Sonnet only for the narrow junior-task carve-out; NEVER Haiku** (CI-enforced).
- **Anti-ruflo:** no fabricated metrics. `confirmed` is **derived**, never a caller-asserted boolean;
  where a fact is unknown, emit `false`/null — never a fabricated `true`/`0`.
- **Family-vocabulary contract pair (a1↔a2):** the external-confirming family set
  `{"GPT","Gemini","Grok"}` (capitalized) is encoded in BOTH `compound-v-epic-arbiter.py`
  (`_EXTERNAL_CONFIRMING`, arbiter:97) and, newly, `compound-v-epic-state.py`'s derivation constant. The
  scripts do not import each other (YAGNI: no shared third module for one tuple), so **casing + membership
  MUST match exactly** and the manifest pins them as a co-reviewed pair. The driver (a3) forwards the
  arbiter's exact strings verbatim (v-epic.md:285).
- **Blocker-category enum (closed):** a CONFIRMED blocker requires the confirming families to agree on the
  **same missing-fact category**, not merely the `blocked_external` label (defends the correlated-oracle
  false-confirm). The closed enum: `credential | external-account | infra | third-party-data |
  legal-approval | human-decision`. A vague/uncategorized blocker, or a category MISMATCH between families,
  downgrades to SUSPECTED (`blocked_needing_human`).
- **Safety (repo-deletion incident class):** no code path may spawn a nested agent with
  `--dangerously-skip-permissions` / `--yolo` / a bypass permission-mode. The agy advisory poll passes
  **none** of those (verified live: `agy --print` answers read-only without the flag). The headless shim
  is **generated text the user installs**; the plugin never loads or runs it.
- **Cross-model gate:** every unit dogfooded, then Codex `gpt-5.6-sol` review to convergence; merge only
  on a clean final integration review (standing delegation).
- **Determinism preserved:** the git-derived scope gate + circuit breakers remain the guardrails; the
  arbiter panel only advises *disposition of a confirmed FAIL* — it can never fabricate a PASS.

## Feature A — `done_with_blockers` via a genuine 2nd external family

### A.1 Why it doesn't work today (grounded)

`compound-v-epic-arbiter.py._aggregate` **already** computes
`confirmed = (len(distinct external families agreeing on blocked_external) >= 2 and retry_n == 0)` over
`_EXTERNAL_CONFIRMING = ("GPT","Gemini","Grok")` (lines 966-974). But the panel only ever polls **Codex
(GPT family)** + validates a driver-supplied **Claude ballot (Claude family)** — so at most **one**
external family is ever present, and `confirmed` is mathematically unreachable. Downstream, three guards
keep `done_with_blockers` inert:

1. `compound-v-epic-state.py` **hard-rejects** `--confirmed true` and `--blocker-confirmed true` (v2.10).
2. `next_feature_autonomous` / `is_terminal` **never emit** `done_with_blockers` (state.py:708).
3. The `/v:epic` §6 `blocked_external` branch always records `confirmed:false`.

### A.2 The design

1. **Arbiter — add a read-only Gemini (`agy`) poll.** In `--classify`, when
   `~/.claude/compound-v-capabilities.json` reports `antigravity.available == true`, poll `agy` as a
   **second, distinct-family advisor** in addition to Codex, mirroring the Codex seam
   (`build_codex_invocation` → new `build_agy_invocation`; `_parse_codex_verdict` → shared/`_parse_agy_verdict`):
   - **Exact safe invocation:** `agy --print <prompt>` under `compound-v-run-with-timeout.py`, cwd = a
     neutral scratch dir, stdin `</dev/null`, `--max-output-bytes` capped. **NO `--dangerously-skip-permissions`,
     no write flags** — advisory only (verified live 2026-07-14: `agy --print` returns a read-only answer,
     exit 0, without the flag). Evidence is **redacted + embedded in the prompt** (same boundary as Codex);
     the poll never relies on agy's own file access.
   - agy prints the answer to **stdout** (no `--output-last-message` file, unlike Codex — `poll_agy` reads
     the supervisor's `--stdout` capture via `_read_capped_regular(..., delete_oversized=True)`, not a
     lastmsg file). Reuse `_parse_codex_verdict`, `redact`, `sanitize_ballot_fields`, `_build_codex_prompt`
     verbatim (DRY — no parallel parse/redaction path; the security-boundary header forbids it).
   - 🔴 **Pin an EXPLICIT resolved Gemini `--model`; resolve family from THAT string; fail-closed.** agy
     1.1.1's live catalog is NO LONGER Gemini-only (it now also serves Claude Opus/Sonnet 4.6 and GPT-OSS
     120B). If the poll omits `--model` and uses agy's ambient default, the ballot's family can resolve to
     Claude (→dropped, `done_with_blockers` silently unreachable) or GPT (→collides with Codex, distinct-set
     collapses to `{GPT}`). So `build_agy_invocation` MUST pass `--model <resolve_agy_model(tier=deep)>`
     (fallback `"Gemini 3.1 Pro (High)"`, mirroring resolve-model `_ANTIGRAVITY`) and `poll_agy` MUST
     resolve `model_family()` from that SAME explicit string. An agy model whose family is unknown/unmappable
     or not in `_EXTERNAL_CONFIRMING` ⇒ **dropped ballot, never assumed Gemini** (fail-closed). Set-dedup at
     arbiter:966-969 already prevents any false *positive*; this rule prevents the false *negative*.
   - Gate: `agy_available(caps)` = `caps["antigravity"]["available"] is True` (single gate, **identity
     check** — a truthy string must not enable it; asymmetric to Codex's double gate by design — agy has no
     exec-flags concept). Test seam symmetric to Codex: `--agy-bin` arg + `COMPOUND_V_ARBITER_AGY_BIN` env +
     `agy_bin=` param; selftest injects a stub, **no live model call**.
   - A poll that errors / times out / fails to parse is a **dropped ballot, never a fabricated vote**.
2. **cursor is NOT a confirming family.** `cursor-agent`'s model is a free-string `auto` whose family is
   non-deterministic — it MAY be added to the panel as an *advisory* ballot later, but it is **excluded
   from CONFIRM** because a mislabeled family could falsely confirm. v2.14 wires **only agy/Gemini** as
   the second confirming family. (Documented limit, not an oversight.)
3. **Arbiter — same-category confirmation (🔴-1 defense).** The ballot schema gains a
   `blocker_category` field (one of the closed enum, or null) alongside the existing free-text `evidence`.
   `_aggregate` no longer confirms on the `blocked_external` *label* alone: `confirming` = the distinct
   external families whose disposition is `blocked_external` **AND** whose `blocker_category` equals the
   agreed category (the category shared by ≥2 external families). `confirmed = len(confirming) >= 2 and
   retry_n == 0`. If the external families name DIFFERENT categories (or any is null/vague), no category is
   shared by ≥2 → `confirmed:false` (SUSPECTED). `families_agreeing` is then **exactly that same-category
   confirming set** — so it already encodes category agreement, keeping the state-side derivation (below) a
   simple length check. `_aggregate` DOES change here (it now reads `blocker_category`); `model_family`,
   `_collapse_same_family`, and the retry/cap masking are untouched.
4. **State — derive `confirmed` from the (category-filtered) families list, never a boolean.** Replace the
   blanket `--confirmed true` hard-reject with a **derivation**: `--update --status blocked` (and
   `--record-disposition blocked_external`) compute `confirmed = (>=2 distinct KNOWN external families in
   --families-agreeing)` **from the families list itself**. Because the arbiter only populates
   `families_agreeing` with same-category agreers (§3), this length check is sound. The raw `--confirmed` /
   `--blocker-confirmed` booleans **stay hard-rejected** — derivation is the ONLY path to `confirmed:true`.
   The agreed `blocker_category` is recorded on the ledger entry via a new optional `--blocker-category`
   arg (for the finish-summary + audit); the two `"confirmed": False` literals (state:1199, 1292) **and the
   reactivation branch** (1208-1218) all switch to the derivation. Fewer than 2 same-category external
   families ⇒ `confirmed:false` (SUSPECTED), exactly as today.
5. **State — new terminal `done_with_blockers` (+ awaiting-final-review pre-terminal, 🔴 finding #10).**
   Extend `next_feature_autonomous` (insert inside `if not pending:`, between the all-done block ~868-882
   and `if blocking_ids:` ~883) and the shared `is_terminal` classifier (add the
   `reason.startswith("done_with_blockers:")` prefix at ~950 — **mandatory, or the watcher never
   terminates**). The `done_with_blockers` branch fires when: every feature is `done` **or** `blocked` with
   a **confirmed** active ledger entry; `blocking_ids` contains **only** `status==blocked` ids (an
   `abandoned`/`halt_feature` feature still forces `blocked_needing_human` — proven work only); no
   `sample_audit_due` is outstanding; and `final_review.status == "passed"`. **Finding #10 — the
   pre-terminal signal:** when the above holds but `final_review` is still *pending*, emit a
   `"running_with_failures: all reachable done, awaiting final_review (done_with_blockers pending)"` reason
   (analogous to the all-done path at state:881) so the driver actually RUNS the final review — otherwise
   `done_with_blockers` is unreachable. A **SUSPECTED** (unconfirmed) blocker with no other runnable work
   still resolves to **`blocked_needing_human`** (unchanged) — only a *confirmed* blocker earns the softer
   terminal. (`_recompute_top_status` need not learn a new value; the driver routes off the
   `next_feature_autonomous` reason, per v-epic §2.)
6. **Driver (`v-epic.md` / `epic-mode.md`) — wire the confirmed path + auto-merge + guardrails.** §6
   `blocked_external` passes the arbiter's `families_agreeing` **and** the agreed `--blocker-category`
   through to `--update --status blocked`. §2 reason dispatch gains a `done_with_blockers` branch (near
   line 193) that routes to **§8 final integration review + `finishing-a-development-branch`** — i.e.
   **auto-merge on a confirmed ≥2-distinct-external-family blocker** (the chosen merge policy), explicitly
   NOT the §7 halt-page. §5/§6 wording updates from "always confirmed:false" to "confirmed derived; pass
   `--families-agreeing` + `--blocker-category` through." **Two cheap guardrails against the reward-hack
   escape-hatch (🔴-2), compatible with auto-merge:** (a) the PASS-integrity sample-audit (§4 of the
   marathon loop) **always over-samples a `confirmed`-blocked feature** — a gamed blocker is higher-stakes
   than a gamed PASS, so it is never in the "not sampled" bucket; (b) `finishing-a-development-branch` /
   the final report **surfaces the blocked remainder explicitly** (feature, category, families-agreeing,
   evidence) for human eyes — never silently dropped. Honest boundary in `epic-mode.md`: a confirmed
   blocker still means *a human must supply the external fact*; `done_with_blockers` reports the epic
   finished **everything it could**, the blocked remainder proven-external (same category) by ≥2 distinct
   external model families — framed as the **minimum defensible bar** (correlated oracles ≈ 1.3–1.5
   effective votes), not strong independent corroboration. Remove the stale "v2.10 never emits
   done_with_blockers" note (epic-mode:165).

### A.3 Acceptance criteria (A)

- With `antigravity.available == true`, `--classify` collects **both** a Codex (GPT) and an agy (Gemini)
  ballot; a `blocked_external` verdict agreed by both **on the SAME `blocker_category`** yields
  `confirmed:true`, `families_agreeing` = `["GPT","Gemini"]` (order-insensitive). Selftest proves it with
  **injected fake backends** (`COMPOUND_V_ARBITER_CODEX_BIN` + a new `COMPOUND_V_ARBITER_AGY_BIN` seam) —
  **no live model call in tests.**
- **Same-category gate:** a selftest where the two families return `blocked_external` with **different**
  `blocker_category` (or one null/vague) yields `confirmed:false` (SUSPECTED) — category agreement, not the
  label, is what confirms.
- **Explicit-Gemini-model assertion:** the built agy argv contains an explicit `--model <Gemini …>`
  (never agy's ambient default) AND **no** `--dangerously-skip-permissions` / `--yolo` / bypass flag.
  A selftest with `COMPOUND_V_ARBITER_AGY_BIN` returning a **GPT-family** model asserts `confirmed==false`
  (family dedups to one `{GPT}` vote — fail-closed, no false positive).
- `compound-v-epic-state.py` records `confirmed:true` **only** when `--families-agreeing` has ≥2 distinct
  known external families; a single-family or Claude-only list ⇒ `confirmed:false`. `--confirmed`/
  `--blocker-confirmed` raw booleans remain rejected (derivation is the only path). The known-external
  constant matches arbiter's `_EXTERNAL_CONFIRMING` casing exactly.
- A state with all features `done`-or-`confirmed-blocked`, no `sample_audit_due`, `final_review passed`,
  and **no abandoned/halt_feature feature** ⇒ `is_terminal` true and `--next --autonomous` emits
  `done_with_blockers: ...`. The same state with `final_review` **pending** emits the awaiting-final-review
  `running_with_failures: ...` signal (not a terminal). A SUSPECTED-blocker state, or one with any
  abandoned feature, still emits `blocked_needing_human`.
- All three scripts' `--selftest` pass under `LANG=C` + Python 3.9. Degrade-safe: `antigravity.available`
  false/absent ⇒ byte-identical v2.11 behavior (Codex-only panel, `done_with_blockers` unreachable, no agy
  call). Checkpoint (non-marathon) path is byte-identical by construction (separate `next_feature` /
  marathon-gated writes).

## Feature B — opt-in headless `launchd`/cron resurrection shim (present-only)

### B.1 Why

Today Tier-2 (`scheduled-tasks`) fires only while the desktop app is **open**, performing one catch-up on
next launch. A user who wants resurrection while the app is closed needs an **OS-level** scheduler. The
memory's honest boundary already says this is a **user-side add-on, never claimed built-in**. v2.14 ships
a **generator** for that add-on — never the daemon itself.

### B.2 The design

- **New script `compound-v-headless-shim.py emit --epic-id E --state S [--interval-min 30] [--os macos|linux]`**
  prints, to stdout, a ready-to-install artifact + runbook:
  - **macOS:** a `launchd` **plist** (`~/Library/LaunchAgents/dev.compound-v.watch.<epic-id>.plist`) whose
    `ProgramArguments` invoke the user's `claude` binary headless on the **Tier-2 resume prompt**
    (`compound-v-epic-watch.py emit-prompt`), on a `StartCalendarInterval` = array of two dicts (`Minute`
    17 / `Minute` 47, no `Hour` key → hourly). **Three launchd traps the generator MUST pre-empt** (else
    the job silently never runs): (i) launchd gives a **minimal PATH** — bake the **absolute** `claude`
    path, resolved via `command -v claude` at emit time (fail the emit with a clear message if unresolved),
    or set an explicit `EnvironmentVariables > PATH`; (ii) **`StandardInPath = /dev/null`** — else `claude
    -p` waits on stdin then exits "no stdin data received"; (iii) **`StandardOutPath`/`StandardErrorPath`**
    to a log file so a silent failure is diagnosable. `RunAtLoad` false (scheduled only).
  - **Linux:** the equivalent `crontab` line `17,47 * * * *` (matches `compound-v-epic-watch.py`'s
    `TIER1_CRON_SCHEDULE`) with an absolute `claude` path and `< /dev/null` redirect.
- **Present-only, gated, never auto-installed** (identical ethos to the `recommend-mcp` / `autoskills`
  recommenders): the script **prints** the artifact + the exact install command for the **user** to run —
  on macOS **`launchctl bootstrap gui/$(id -u) <plist>`** primary (with legacy `launchctl load` noted as a
  fallback for old macOS — `load`/`unload` are soft-deprecated on modern macOS in favor of
  `bootstrap`/`bootout`), teardown `launchctl bootout gui/$(id -u)/<label>`; on Linux `crontab -e` / append.
  The plugin **never** shells out to `launchctl`/`crontab` and never runs the generated agent (asserted:
  no such subprocess in the code path).
- **Safe non-interactive posture — `--permission-mode dontAsk` + a curated read-mostly `--allowedTools`
  allowlist** (the chosen posture). A headless `claude -p` under the user's *interactive* posture **stalls**
  on the first tool prompt (no TTY); the naïve "fix" a frustrated user reaches for —
  `--dangerously-skip-permissions` — is (a) the flag that **deleted this repo** in a live incident and (b)
  not even the correct headless flag (it still prompts on first use). So the emitted command uses
  `--permission-mode dontAsk` (runs read-only + the allowlist, **refuses** everything else — never blocks,
  never bypasses) with an `--allowedTools` allowlist scoped to the resume driver's needs (read/grep/status
  + the specific resume-driver command). The runbook carries a **prominent DO-NOT block:** *"Never add
  `--dangerously-skip-permissions` / `--yolo` to this plist — a headless bypass agent with no human present
  has deleted a repository in this project's own history. If resurrection stalls, that is the safety system
  working; widen `--allowedTools` deliberately, never bypass."*
- **Honest boundary in the runbook (precise):** removes the app-open dependency; **launchd fires on the
  `:17/:47` cadence while awake and runs once on wake-from-sleep to catch up a missed slot (missed slots
  coalesce to one run — effectively one catch-up per wake).** It does **NOT** run while the Mac is powered
  off (missed slots lost until the next live slot) or during sleep itself; a `gui/$UID` LaunchAgent also
  only loads once the user is logged into the GUI session (a FileVault pre-login screen is a dead zone).
  Genuine always-on needs remote infra. `max_resume_count` still bounds total fires.

### B.3 Acceptance criteria (B)

- `emit --os macos` prints a plist that (a) is valid XML, (b) contains the epic id and the resume-prompt
  invocation, (c) contains **no** `--dangerously-skip-permissions` / `--yolo`, (d) **contains
  `--permission-mode dontAsk` + a non-empty `--allowedTools` allowlist** (positive assertion — "safe" must
  not silently mean "stalls forever"), (e) bakes an **absolute** `claude` path and `StandardInPath`
  `/dev/null`, (f) targets a `LaunchAgents` path and prints **`launchctl bootstrap gui/$(id -u) <plist>`**
  (+ `bootout` teardown) as a **user** step, not an executed one.
- `emit --os linux` prints a crontab line (`17,47 * * * *`) with an absolute `claude` path, `< /dev/null`,
  the same `dontAsk`+allowlist posture, and the same no-dangerous-flag guarantee.
- The emit **fails with a clear message** if `claude` cannot be resolved to an absolute path (never emits a
  bare `claude` that launchd's minimal PATH would fail to find).
- `--selftest` covers both OSes, the no-dangerous-flag assertion, the positive `dontAsk`+allowlist
  assertion, the absolute-path + `/dev/null`-stdin assertions, and a malformed-arg refusal. The script
  **never** shells out to `launchctl`/`crontab` (asserted: no such subprocess in the code path).
- Wired into `/v:onboard` doctor output and/or the `/v:epic` watch honesty note as a **present-only** pointer.
