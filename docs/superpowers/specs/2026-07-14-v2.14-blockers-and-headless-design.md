# v2.14.0 — Confirmed Blockers (2nd external family) + Headless Resurrection Shim + Doc Refresh

**Goal:** Give Compound V's Epic Autonomous mode a genuinely *confirmed* blocker terminal
(`done_with_blockers`) backed by **two distinct external model families**, ship an **opt-in headless
`launchd`/cron shim** so a marathon can resurrect while the desktop app is closed, and **refresh the 5
stale architecture-doc targets** — all as one release, `2.14.0`.

**Architecture:** Three disjoint workstreams. **A** extends the existing v2.10 arbiter panel
(`compound-v-epic-arbiter.py`) to poll a **second external family** (Gemini via `agy`, read-only
advisory) alongside Codex (GPT), un-gates the *derived* `confirmed` path in `compound-v-epic-state.py`,
and adds the `done_with_blockers` terminal to the state spine + `/v:epic` driver. **B** adds a
**present-only generator** (`compound-v-headless-shim.py`) that prints a macOS `launchd` plist / Linux
cron entry + runbook — the user installs it; the plugin never `launchctl load`s anything and never emits
`--dangerously-skip-permissions`. **C** runs `/v:onboard --refresh` **last** (after A+B land) so the
regenerated citations reflect the final code.

**Tech Stack:** Python 3.9-safe stdlib only (no `match`, no `X | Y` unions); Bash (`bash -n` clean);
the existing `compound-v-run-with-timeout.py` supervisor; the existing capability-detection files
(`~/.claude/compound-v-capabilities.json`, `.claude/compound-v.json`). No new dependencies, no services.

## Global Constraints

- **Opus by default; Sonnet only for the narrow junior-task carve-out; NEVER Haiku** (CI-enforced).
- **Anti-ruflo:** no fabricated metrics. `confirmed` is **derived**, never a caller-asserted boolean;
  where a fact is unknown, emit `false`/null — never a fabricated `true`/`0`.
- **Safety (repo-deletion incident class):** no code path may spawn a nested agent with
  `--dangerously-skip-permissions` / `--yolo` / a bypass permission-mode. The agy advisory poll passes
  **none** of those (verified live: `agy --print` answers read-only without the flag). The headless shim
  is **generated text the user installs**; the plugin never loads or runs it.
- **Cross-model gate:** every unit dogfooded, then Codex `gpt-5.6-sol` review to convergence; merge only
  on a clean final integration review (standing delegation).
- **Determinism preserved:** the git-derived scope gate + circuit breakers remain the guardrails; the
  arbiter panel only advises *disposition of a confirmed FAIL* — it can never fabricate a PASS.

## Feature A — `done_with_blockers` via a genuine 2nd external family

### A.1 Why it doesn't work today (grounded)

`compound-v-epic-arbiter.py._aggregate` **already** computes
`confirmed = (len(distinct external families agreeing on blocked_external) >= 2 and retry_n == 0)` over
`_EXTERNAL_CONFIRMING = ("GPT","Gemini","Grok")` (lines 966-974). But the panel only ever polls **Codex
(GPT family)** + validates a driver-supplied **Claude ballot (Claude family)** — so at most **one**
external family is ever present, and `confirmed` is mathematically unreachable. Downstream, three guards
keep `done_with_blockers` inert:

1. `compound-v-epic-state.py` **hard-rejects** `--confirmed true` and `--blocker-confirmed true` (v2.10).
2. `next_feature_autonomous` / `is_terminal` **never emit** `done_with_blockers` (state.py:708).
3. The `/v:epic` §6 `blocked_external` branch always records `confirmed:false`.

### A.2 The design

1. **Arbiter — add a read-only Gemini (`agy`) poll.** In `--classify`, when
   `~/.claude/compound-v-capabilities.json` reports `antigravity.available == true`, poll `agy` as a
   **second, distinct-family advisor** in addition to Codex, mirroring the Codex seam
   (`build_codex_invocation` → new `build_agy_invocation`; `_parse_codex_verdict` → shared/`_parse_agy_verdict`):
   - **Exact safe invocation:** `agy --print <prompt>` under `compound-v-run-with-timeout.py`, cwd = a
     neutral scratch dir, stdin `</dev/null`, `--max-output-bytes` capped. **NO `--dangerously-skip-permissions`,
     no write flags** — advisory only (verified live 2026-07-14: `agy --print` returns a read-only answer,
     exit 0, without the flag). Evidence is **redacted + embedded in the prompt** (same boundary as Codex);
     the poll never relies on agy's own file access.
   - agy prints the answer to **stdout** (no `--output-last-message` file, unlike Codex) — parse the
     verdict JSON from stdout, capped.
   - Family is resolved through the existing `model_family()` map (`agy` deep model → **Gemini**). A poll
     that errors / times out / fails to parse is a **dropped ballot, never a fabricated vote** (existing
     `_aggregate` rule).
2. **cursor is NOT a confirming family.** `cursor-agent`'s model is a free-string `auto` whose family is
   non-deterministic — it MAY be added to the panel as an *advisory* ballot later, but it is **excluded
   from CONFIRM** because a mislabeled family could falsely confirm. v2.14 wires **only agy/Gemini** as
   the second confirming family. (Documented limit, not an oversight.)
3. **State — derive `confirmed`, don't trust a boolean.** Replace the blanket `--confirmed true`
   hard-reject with a **derivation**: `--update --status blocked` (and `--record-disposition
   blocked_external`) compute `confirmed = (>=2 distinct KNOWN external families in --families-agreeing)`
   **from the families list itself**, never from a raw caller boolean. A caller still cannot *assert*
   confirmation — it is a function of the (arbiter-produced, audit-frozen) `families_agreeing`. Fewer than
   2 distinct external families ⇒ `confirmed:false` (SUSPECTED), exactly as today.
4. **State — new terminal `done_with_blockers`.** Extend `next_feature_autonomous` + the shared
   `is_terminal` classifier: when every feature is `done` **or** `blocked` with a **confirmed** ledger
   entry, no `sample_audit_due` is outstanding, and `final_review.status == "passed"`, the epic resolves
   to **`done_with_blockers`** (a *successful* terminal — the reachable work is finished, the impossible
   work is confirmed-isolated with proof). An epic with any **SUSPECTED** (unconfirmed) blocker and no
   other runnable work still resolves to **`blocked_needing_human`** (unchanged) — only a *confirmed*
   blocker earns the softer terminal.
5. **Driver (`v-epic.md` / `epic-mode.md`) — wire the confirmed path.** §6 `blocked_external` passes the
   arbiter's `families_agreeing` through to `--update --status blocked`; the terminal branch (§7/§8) routes
   `done_with_blockers` to the final integration review + `finishing-a-development-branch` (like `done`),
   distinct from the `blocked_needing_human` halt-page. Honest boundary documented: a confirmed blocker
   still means *a human must supply the external fact* — `done_with_blockers` reports the epic finished
   **everything it could**, with the blocked remainder proven-external by ≥2 independent model families.

### A.3 Acceptance criteria (A)

- With `antigravity.available == true`, `--classify` collects **both** a Codex (GPT) and an agy (Gemini)
  ballot; a `blocked_external` verdict agreed by both yields `confirmed:true`, `families_agreeing`
  containing `["GPT","Gemini"]` (order-insensitive). Selftest proves it with **injected fake backends**
  (`COMPOUND_V_ARBITER_CODEX_BIN` + a new `COMPOUND_V_ARBITER_AGY_BIN` seam) — **no live model call in tests.**
- The agy invocation argv in a selftest contains **no** `--dangerously-skip-permissions` / `--yolo` /
  bypass flag (asserted on the built argv).
- `compound-v-epic-state.py` records `confirmed:true` **only** when `--families-agreeing` has ≥2 distinct
  known external families; a single-family or Claude-only list ⇒ `confirmed:false`. `--confirmed`/
  `--blocker-confirmed` raw booleans remain rejected (derivation is the only path).
- A state with all features `done`-or-`confirmed-blocked` + `final_review passed` ⇒ `is_terminal` true and
  `--next --autonomous` emits `done_with_blockers: ...`; a SUSPECTED-blocker state still emits
  `blocked_needing_human`.
- All three scripts' `--selftest` pass under `LANG=C` + Python 3.9. Degrade-safe: `antigravity.available`
  false/absent ⇒ byte-identical v2.11 behavior (Codex-only panel, `done_with_blockers` unreachable, no agy call).

## Feature B — opt-in headless `launchd`/cron resurrection shim (present-only)

### B.1 Why

Today Tier-2 (`scheduled-tasks`) fires only while the desktop app is **open**, performing one catch-up on
next launch. A user who wants resurrection while the app is closed needs an **OS-level** scheduler. The
memory's honest boundary already says this is a **user-side add-on, never claimed built-in**. v2.14 ships
a **generator** for that add-on — never the daemon itself.

### B.2 The design

- **New script `compound-v-headless-shim.py emit --epic-id E --state S [--interval-min 30] [--os macos|linux]`**
  prints, to stdout, a ready-to-install artifact + runbook:
  - **macOS:** a `launchd` **plist** (`~/Library/LaunchAgents/dev.compound-v.watch.<epic-id>.plist`) whose
    `ProgramArguments` invoke the user's `claude` binary headless on the **Tier-2 resume prompt**
    (`compound-v-epic-watch.py emit-prompt`), on a `StartCalendarInterval` matching the `:17/:47` cadence.
  - **Linux:** the equivalent `crontab` line.
- **Present-only, gated, never auto-installed** (identical ethos to the `recommend-mcp` / `autoskills`
  recommenders): the script **prints** the artifact + the exact `launchctl load` / `crontab -e` command
  for the **user** to run. The plugin **never** runs `launchctl load`, `crontab`, or the generated agent.
- **No dangerous flag baked in.** The emitted `claude -p` command uses the user's normal permission
  posture; the runbook states plainly that headless resurrection requires the user to decide a trust
  posture, and that the shim deliberately does **not** embed `--dangerously-skip-permissions`.
- **Honest boundary in the runbook:** removes the app-open dependency but **still does not run while the
  laptop is asleep/closed**; genuine always-on needs remote infra. `max_resume_count` still bounds it.

### B.3 Acceptance criteria (B)

- `emit --os macos` prints a plist that (a) is valid XML, (b) contains the epic id and the resume-prompt
  invocation, (c) contains **no** `--dangerously-skip-permissions` / `--yolo`, (d) targets a
  `LaunchAgents` path and prints the `launchctl load` command as a **user** step, not an executed one.
- `emit --os linux` prints a crontab line with the same cadence + prompt and the same no-dangerous-flag guarantee.
- `--selftest` covers both OSes, the no-dangerous-flag assertion, and a malformed-arg refusal. The script
  **never** shells out to `launchctl`/`crontab` (asserted: no such subprocess in the code path).
- Wired into `/v:onboard` doctor output and/or `/v:epic` watch honesty note as a **present-only** pointer.
