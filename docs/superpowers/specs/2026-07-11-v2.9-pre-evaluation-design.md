# v2.9 — Pre-Evaluation & Proportionate Fast-Path — Design Spec

**Recon:** none (this design was grounded by a five-pass research corpus + an independent
cross-model design critique instead of a Trigger-0 recon; see References).

**Status:** design approved section-by-section (2026-07-11). Next: `writing-plans` →
disjoint partition map → manifest → dispatch → three-pass Review Gate → Codex build rounds.

**Goal:** Add a fast, cheap **Pre-Evaluation** stage before Trigger 0 that scores each request
on two axes (difficulty, impact) from tiered deterministic evidence, and — only when a change is
provably trivial and low-impact — OFFERS the user a proportionate fast-path instead of the full
brainstorm → 3 pre-flights → partition → 3-pass-review pipeline. Trivial fixes stop being a
Nolan odyssey; nothing risky ever silently skips ceremony.

**Architecture:** A new pre-brainstorm stage writing a separate machine-readable record
(`pre-eval.json`), a deterministic scoring truth-table (no raw LLM magnitude), a bounded
read-only localization step, three new state-machine states with a real escalation path, a
fast-path skip-contract that keeps the manifest + scope-gate + a proportionate review, and
onboarding extensions that manufacture the taxonomy + churn signals the scorer reads.

**Tech Stack:** Python stdlib (scoring engine, localization, churn), YAML (taxonomy), JSONL
(triage-outcomes, append-only), the existing manifest/state-machine/scope-gate substrate, the
existing FTS5 V-memory index (ADR indexing), one `light`-tier model call (Tier 3 only, rare).

---

## Global Constraints

- **Model policy:** Opus by default; `light`-tier T3 classify may run Sonnet; **never Haiku**
  (enforced by `lint-frontmatter.py` + CI). Reviewers are **always** `deep`/opus (manifest
  invariant #4) — unchanged.
- **No fabricated metrics** (anti-ruflo): the displayed 1-10 is a band-midpoint label, never a
  token/cost estimate; no cost numbers anywhere in pre-eval output.
- **External-launch invariant:** any external CLI (T3 if routed to codex, churn is pure git) goes
  through `scripts/compound-v-run-with-timeout.py` with `stdin </dev/null`.
- **Git-derived enforcement, never model self-report:** the scope gate and the post-hoc diff
  re-classification read git, not any worker's claim. Unchanged.
- **Commit-discipline (v2.6.4):** `pre-eval.json`, `triage-outcomes.jsonl`, `churn-cache.json`,
  the taxonomy, and ADR docs are **written AND committed** — an uncommitted artifact vanishes on
  `git worktree remove` and never indexes into V-memory.
- **Recon boundary preserved:** Pre-Evaluation is a **routing/triage** artifact; recon
  (`phase-0-recon.md`) stays evidence-only, five-section, never a routing input. They are separate
  records. Recon's contract is not modified.

## Iron Invariants (honesty constraints — not scope-negotiable)

1. **No raw LLM magnitude.** The LLM only ever classifies into pre-declared bands; the score is
   assembled by deterministic logic. (Evidence: identical inputs scored 6.8/5.2/7.5 across runs.)
2. **Localization before any `low`.** A `low` verdict is impossible until a bounded read-only step
   resolves real paths/tokens/fan-out. "make button X red" may be a global design token or an a11y
   contrast state — discovered before the decision, not after.
3. **Tier 2 self-gates on `min_sample_count` + counterfactual guard.** Historical outcomes may only
   lower ceremony once enough *fast-path-taken* outcomes have accrued and calibrated. Legacy
   full-pipeline successes are counterfactual and may only ESCALATE, never fast-path. At launch,
   with no data, Tier 2 is escalation-only by construction.
4. **Request-level score only OFFERS, never auto-routes.** (Evidence: text-only prediction AUC
   0.52-0.57 vs structural 0.957; every production triage system keeps a human-confirm step.)
5. **Fail-closed everywhere.** Any ambiguity, missing data, tier disagreement, token-budget
   overrun, or `unknown` axis → `FULL_PIPELINE`. Pre-eval can only ever *save* work on the proven
   trivial path; it never fails open.
6. **The scope gate is never skippable, and the prediction is never trusted once a diff exists.**
   Post-hoc the real diff is re-classified; a violation escalates to the full pipeline.

Trigger-1 pre-flights (1A code-archaeology, 1B domain, 1C doc-validator, 2026-07-11) read the
actual code and corrected three factually-wrong premises in the first draft. These are binding:

  passes today. So enforcing the skip-record *shape* (`skipped`, `reason`, `localization`,
  `taxonomy_version`) is **net-new validation to ADD**, and the sentinel MUST be emitted in
  **block YAML** (the no-PyYAML `_mini_yaml` fallback mis-parses flow `{}` mappings — add a
  `_mini_yaml` nested-mapping self-test fixture).
- **AC-2 (was §1/§2 wrong): the run dir / `run-id` does NOT exist at pre-brainstorm time.**
  It is created post-plan (`v-orchestrate` Step 3 / `parallel-dispatcher` Step 0). So
  `pre-eval.json` **cannot** live at `execution/<run-id>/` when PRE_EVAL runs.
  **Resolution:** PRE_EVAL writes to a pre-run path `docs/superpowers/pre-eval/<request-slug>.json`
  (slug derived from the request, not the not-yet-existing plan); once the run dir exists, the
  run references that path (and MAY copy it into `execution/<run-id>/` for the audit trail).
- **AC-3 (was §6 wrong): `triage-outcomes.jsonl` is strictly append-only — NO back-fill.**

  a `run_id` does not exist at prediction time, so the join key MUST be the write-once `pre_eval_id`,
  minted before any run): `{event:"predicted", pre_eval_id, …}` at `PRE_EVAL_DONE`; then, if the
  request becomes a run, `{event:"bind", pre_eval_id, run_id}` when the run dir is created; then
  `{event:"actual", pre_eval_id, run_id, …}` at `MERGED`/`ESCALATION`. Tier 2 joins on `pre_eval_id`
  at read time (and reaches `run_id` via the `bind` event). Reuse `update-memory.py` `append_line`
  discipline (forbidden-basename guard, makedirs, never rewrite/sort/dedupe). Duplicate/out-of-order

  in place.** `/v:resume` replays a fixed job set. **Resolution:** on `ESCALATION_REQUIRED`, mint a
  **new run-id** for the full pipeline and link it from the fast-path run's `state.json`
  (`escalated_to: <new-run-id>`); the fast-path patch is preserved as evidence under the original
  run. No in-place rewrite.
- **AC-5 (H7): the post-hoc re-classifier is a SIBLING analyzer, never an extension of
  `compound-v-scope-check.py`** (a hardened name-only gate with reproduced-exploit selftests).
  Reuse scope-check's changed-path SET for "touches a sensitive path"; get size from a separate
  `git diff --numstat`; structural/control-flow features are net-new. The scope gate stays
  name-only and never reads diff content.
- **AC-6 (H8): PRE_EVAL is description-driven and UNENFORCEABLE** — the only pre-brainstorm seam is
  the reminder-only `PreToolUse(Skill)` hook `brainstorm-trigger0-nudge.sh` (it cannot block or run
  Python). PRE_EVAL is exactly as weak as Trigger 0. This is safe **only** because Iron-Invariant #5
  (fail-closed → `FULL_PIPELINE`) means a missed PRE_EVAL degrades to the normal pipeline. Do NOT
  claim PRE_EVAL is enforced; keep any hook change reminder-only + `jq`-guarded + platform-branched.
- **AC-7 (favorable + round-2 correction): new phases are prose-only** — no script hard-codes the
  phase enum. **`PRE_EVAL_DONE` is a status field in the write-once pre-eval RECORD, not a
  `state.json` phase** (there is no run dir / `state.json` at prediction time). `FASTPATH_DISPATCHED`
  and `ESCALATION_REQUIRED` ARE `state.json` phases (a run exists by then). On an accepted fast-path,
  the linked run initializes at `FASTPATH_DISPATCHED`; on decline/full-pipeline the run initializes
  at the normal first phase. `state-machine.md`, `commands/v-status.md`, `commands/v-resume.md`, and
  `agents/parallel-dispatcher.md` MUST learn these + the escalation transitions, or status/resume
  render/behave undefined.
- **AC-13 (round-2 crit): fast-path manifest cross-artifact binding is validator-enforced.** The
  sole `write_allowed` literal path MUST equal `localization.resolved_paths[0]`; `pre_eval_id`,
  `FASTPATH_ELIGIBLE` decision, `taxonomy_sha`/digest, and the localization content digest MUST match
  across the manifest, the pinned pre-eval record, and the localization artifact. A mismatch fails
  validation (tampering fixtures required) — otherwise a manifest could cite a safe CSS localization
  while authorizing a different file the scope gate would then happily enforce.
- **AC-14 (round-2 crit): an accepted fast-path offer is MATERIALIZED by a dedicated owner** into
  committed run artifacts (minimal spec/plan stubs, block-YAML audit skip-records, the fast-path
  manifest, initial `state.json` at `FASTPATH_DISPATCHED`, the captured implementer prompt, an
  immutable taxonomy snapshot + digest), in a defined commit order — no existing task silently
  assumes the manifest appears.
- **AC-15 (round-2 high): escalation is crash-consistent** via an idempotent two-phase protocol —
  commit patch+baseline evidence first, derive a deterministic child run-id, create+commit the child,
  then commit the parent `escalated_to` link; `/v:resume` reconciles every partial state and
  discovers an existing child before minting another.
- **AC-16 (round-2 high): content-pattern regex is bounded** — a documented safe subset (no nested
  quantifiers), deterministically validated, AND matching runs in a killable subprocess through the
  timeout supervisor (Python 3.9 `re` has no match timeout); an adversarial nested-quantifier fixture
  must terminate within a fixed bound. Typed structural analysis is real for Python (`ast`); JS/TS/
  Go/Ruby escalate fail-closed unless the change is provably trivial (comment/string/whitespace) or a
  language parser is present.
- **AC-8 (1B DC-3): impact is decided on what a change semantically IS, not only where its file
  lives.** The taxonomy gains a **content-pattern** side (not only path-patterns) covering four
  tiny-diff-high-blast-radius surfaces a dir scan misses: **legal/compliance copy**, **i18n
  interpolation placeholders** (`{{var}}`/`%s`), **feature-flag definitions/default flips**, and
  **config/constant literals** (timeouts, pricing, quotas, rate limits). Each carries a cited
  incident. Localization flags these by content; `/v:onboard`'s taxonomy-draft OFFERS them per-repo.
- **AC-9 (1B DC-1/DC-2): the offer stays folded into the ONE recon/clarify interaction, never a
  standalone screen** (per-request prompting trains rubber-stamping → de-facto auto-route,
  violating Iron-Invariant #4). Deterministic tiers must resolve inside the ~1s flow ceiling; the
  T3 `light` call stays off the common path and shows a "checking…" affordance if it can exceed ~1s.
- **AC-10 (1C corrections): tool-invocation reality.** Use `python3 -m py_compile` (not
  `py_compile`); gate every parse-checker on binary-present ∧ project-manifest-present, degrade
  never crash (`tsc`/`cargo`/`php`/`go` not guaranteed); localization degrades `rg → git grep →
  grep` (no hard ripgrep dep); churn uses `git log --no-merges --numstat --format='%x01…'` and MUST
  handle the `-\t-` binary sentinel + separated revert/fix grep (`git log -i -E --grep=revert

  model); taxonomy validator copies the soft-PyYAML+stdlib pattern (no hard `import yaml`).

- **AC-11 (1B R1, product decision — IN v2.9): remember-my-choice per taxonomy-category.**
  A developer who accepts the fast-path for a category (e.g. `css-only`) MAY opt to not be re-asked
  for that category. Stored as an explicit, revocable per-category opt-in in `.claude/compound-v.json`
  (`pre_eval.remember: {css-only: "fastpath", …}`). **This does NOT weaken Iron-Invariant #4:** it
  is an explicit one-time human opt-in per category (the Linear per-property precedent), not a
  silent auto-route — the fail-closed overrides (sensitive path, shared-token, a11y, churn-hot,
  tier-disagreement, post-hoc diff escalation) STILL fire on every request regardless of a
  remembered choice. Revoked via `/v:init` or by editing config. Default: not remembered (ask).
- **AC-12 (1B R2/Q3, product decision — IN v2.9): `/v:status` shows fast-path precision +
  escalation-rate.** Computed from the `triage-outcomes.jsonl` `predicted`↔`actual` join
  (git-derived actuals only — no fabricated metric, anti-ruflo safe). Makes accuracy drift
  human-visible (an early signal the taxonomy needs tuning). A running figure, honestly labelled
  with its sample size; "insufficient samples" when below a floor, never a made-up number.

**Reuse targets (no new copies):** `_seg_is_literal`/`glob_match` (`validate-manifest.py`),

## 1. Pipeline placement, records, and states

New stage, first in line:

```
request
  └─► PRE-EVAL (new) ──► Trigger 0 recon ──► brainstorming ──► Trigger 1 ──► partition ──► ...
        ├─ bounded read-only LOCALIZATION (resolve real files/tokens/fan-out)
        ├─ Tier 1 taxonomy · Tier 2 history (min_sample-gated) · Tier 3 light LLM-classify (rare)
        ├─ churn signal (escalation-only) from onboarding cache
        └─ writes SEPARATE machine-readable pre-eval.json (+ triage-outcomes.jsonl)
```

**Records:**
- `docs/superpowers/pre-eval/<request-slug>.json` — control-plane state (see §2 for shape). **Not**
  `execution/<run-id>/` — that dir doesn't exist yet at pre-brainstorm time (AC-2). Once the run dir
  exists it references (and MAY copy in) this file for the audit trail.
- `docs/superpowers/memory/triage-outcomes.jsonl` — append-only three-event stream (`predicted` →
  `bind` → `actual`), joined on the write-once `pre_eval_id` (AC-3; §6).
- Recon's own doc + `recon-outcomes.jsonl` are untouched.

**New state-machine states** (extend the linear `SPEC_READY→…→MERGED` + `BLOCKED`):

| State | Reached when |
|---|---|
| `PRE_EVAL_DONE` | Score computed, `pre-eval.json` written **and committed**. Precedes `SPEC_READY`. |
| `FASTPATH_DISPATCHED` | Fast-path eligible, user accepted; the single-job fast-path manifest dispatched. |
| `ESCALATION_REQUIRED` | Post-hoc diff re-classification failed; patch preserved as evidence, pipeline rejoins the full path. |

Transitions:

```
PRE_EVAL_DONE ─(eligible ∧ accept)─► FASTPATH_DISPATCHED ─► [scope-gate + parse/test]
    ├─(diff clean)────► REVIEWED (1 combined pass) ─► MERGED
    └─(diff violates)─► ESCALATION_REQUIRED ─► PREFLIGHT_DONE ─► PARTITION_VERIFIED ─► … (normal)
PRE_EVAL_DONE ─(not eligible ∨ decline ∨ off)─► SPEC_READY ─► … (normal)
```

`state.json` is written after every transition; `pre-eval.json` is committed before advancing
(same commit-discipline as recon — else worktree cleanup deletes it and `/v:status` goes blind).

## 2. Scoring truth-table (deterministic; no raw LLM magnitude)

Two axes computed **separately**, each → band `{low, medium, high, unknown}`. Within an axis:
**conservative-max across tiers, never average down** — a single strong high signal must not be
diluted. "Coefficients" survive as per-tier authority weighting; "multi-dimensional" survives as
the two axes never merging into one number.

### Layer A — hard overrides (fail-closed, ordered; first match → `FULL_PIPELINE`, zero further cost)

| # | Condition | Rationale |
|---|---|---|
| 1 | localization = `failed` ∨ `ambiguous` | paths unknown → cannot judge |
| 2 | any resolved path matches the **sensitive path-list** (taxonomy) | auth/payments/PII/a11y/migrations/infra |
| 3 | `is_shared_token` ∨ `is_generated` ∨ `is_a11y_state` | "button" = global token / contrast |
| 4 | semantic-vs-path disagreement (T1 `low` but T3 `user-facing-major`, or converse) | tier disagreement escalates |
| 5 | `churn.hot` on any resolved path | churn ESCALATES only; low/insufficient never lowers |
| 6 | any axis = `unknown` | no signal → full pipeline |

Most non-trivial requests hit an override with **zero LLM calls**.

### Layer B — positive fast-path gate (only if no override fired)

`FASTPATH_ELIGIBLE` ⟺ `difficulty == low` ∧ `impact == low` ∧ `fan_out ≤ threshold` ∧
**exactly one literal normalized path** (manifest single-path partition rule — no glob metachar,
not shared/generated/config/migration).

### Band composition (conservative-max)

**T3 total truth table (deterministic — every enum → both axes; no `low/med` ambiguity, round-3 fix):**

| T3 category | difficulty | impact |
|---|---|---|
| `plumbing` | low | low |
| `user-facing-minor` | medium | medium |
| `user-facing-major` | high | high |
| `unknown` | unknown | unknown |

**Difficulty:** T1 taxonomy is primary. T2 (`matched ∧ n≥min_sample ∧ healthy`) *corroborates* low
but never independently creates it pre-calibration; `unhealthy` raises. T1-unclassified ∧
T2-insufficient → T3 per the table above. Tier conflict → higher band.

**Impact** (weaker text proxies → asymmetric): T1 taxonomy primary and most authoritative; T3
impact is low-confidence and may only **raise**, never lower below T1.

**T3 may never manufacture fast-path eligibility without T1 safety coverage** (round-3 fix): a `low`
from T3 alone, with NO taxonomy loaded (so no sensitive-path / content-pattern protection exists),
does NOT satisfy Layer-B. Absent/malformed/unavailable taxonomy or its pinned snapshot →
**unconditional `FULL_PIPELINE`** (below).

### Derived 1-10 (display only)

`low→1-3, med→4-6, high→7-10`; show band-midpoint (or per-tier max) **per axis**. A label computed
*after* the decision — never the gate.

### Declared missing-data behavior

| Missing | Behavior |
|---|---|
| taxonomy file / snapshot absent, malformed, or unavailable | **unconditional `FULL_PIPELINE`** (round-3 fix: without the sensitive-path + content-pattern protections there is no way to *prove* a change is safe; T3 alone must not manufacture eligibility) |
| churn cache | churn signal absent (never escalates, never lowers — safe) |
| triage-outcomes empty / n<min_sample | T2 insufficient (escalation-only) |
| localization fails | override #1 → FULL_PIPELINE |
| token-budget overrun | abort pre-eval → FULL_PIPELINE |

### `pre-eval.json` shape (records everything needed to reproduce + learn)

`{run_id, request_slug, taxonomy_version, ts, difficulty:{band, display}, impact:{band, display},
tiers_signalled:[…], localization:{resolved_paths, fan_out, flags, confidence},
override_fired:<id|null>, decision:<FASTPATH_ELIGIBLE|FULL_PIPELINE>, min_sample_status}`.
`confidence` = signalled tiers + localization_confidence, shown as **evidence**, never a gating
number.

## 3. Cost/speed budget (Pre-Eval must be near-free by construction)

Tier order guarantees common cases exit with **zero model calls**: localization (ripgrep + glob +
git), T1 (YAML lookup), churn (cache lookup), T2 (local FTS5 query), Layer-A override — all
deterministic. **T3 is the only model call, reached only when `T1 unclassified ∧ T2 insufficient`.**

Three budget rules:
1. **Localization is bounded** — not mini-archaeology. Resolve symbol/selector/token via
   ripgrep+glob, hard cap on files inspected + timeout. Cannot resolve within cap → `ambiguous` →
   override #1 → full pipeline (degrades to "just do it normally," never spends more).
2. **T3 = `light` tier** (Sonnet, never Haiku), one classify call, tiny input (request text +
   resolved paths + taxonomy categories), bounded enum output, no thinking budget.
3. **Declared token-cap on the whole stage.** Overrun → abort → `FULL_PIPELINE` (fail-safe). Worst
   case: pre-eval costs its bounded budget, then the normal pipeline runs — the same pipeline that
   would have run anyway.

## 4. Fast-path skip-contract

**Skipped:** heavy brainstorm (→ minimal confirm); 1A (localization already did archaeology-lite);
1B/1C via **validated sentinel skip-records**; multi-job partition (single literal path → auto
single-job manifest).

**Never skipped:** manifest materialization + commit; git-diff scope gate; the test floor; a
**proportionate** review (not zero).

**Sentinel audits.** The manifest requires only that `audits` be a non-empty dict (AC-1), so a
skip-record already passes structurally. Fast-path writes a tiny **block-YAML** skip-record per
audit: `{skipped:true, reason:fastpath, localization:<path>, taxonomy_version}` — auditable, not a
silent null. Enforcing the skip-record *shape* is **net-new** validation to ADD to
`compound-v-validate-manifest.py` (+ a `_mini_yaml` nested-mapping self-test).

**Review-pass matrix.** Reviewer-invariant (reviewers ⇒ deep/opus) is untouched — savings come from
skipping brainstorm + 3 pre-flights + partition, NOT from a cheaper reviewer. The review itself:
INTEGRATION pass is vacuous (single job, no seams) → auto-pass with recorded rationale; SPEC +
QUALITY → **one combined diff-scoped pass** (Opus, honest, single-pass on a tiny diff).

**Test floor** (from the no-tooling ladder research): tier-1 real tests if configured → else tier-2
language parse-check on touched file (`py_compile`/`node --check`/`tsc --noEmit`/`go build`/`ruby
-c`/`php -l`) → else tier-3 one cheap diff-read → else full review. Scope-gate + this floor are the
non-skippable minimum.

**Manifest schema changes.** New optional `fast_path:{eligible, pre_eval:<path>,
localization:<path>, taxonomy_version}`; `audits` accept skip-records; single-job exception in the
validator — **exactly one literal normalized path**, no glob metachar, not
shared/generated/config/migration (Codex's rule verbatim).

**Post-hoc escalation.** Once the diff exists, re-classify it with the strong diff-lane signal
(structural features; AUC 0.957 in the MSR-2026 evidence). Diff touches a sensitive path / exceeds a
size threshold / adds control-flow or signature changes → `ESCALATION_REQUIRED`: preserve the patch
as evidence, run the skipped 1A/1B/1C + planning, then revalidate/reimplement → merge. Real evidence
overrides a wrong prediction.

## 5. Onboarding extensions (present-then-confirm, human-gated, never auto-applied)

Into the existing 9-step `/v:onboard` (DIAGNOSE drafts → GATE per-section diff → WRITE/COMMIT/INDEX):

1. **Taxonomy-draft → `.claude/compound-v-impact-taxonomy.yaml`.** Scan directory/module structure +
   stack; draft `path-pattern → (difficulty_band, impact_band)` rows + a **sensitive path-list**
   section (auth/payments/PII/a11y/migrations/infra globs from the real repo structure — the
   path-list that did not previously exist) + `taxonomy_version`. CSS rows carry the
   "shared-token/a11y caught by localization" caveat so override #3 fires rather than a naive `low`.
   Human edits/approves at GATE. Never auto-applied.
2. **Normalized churn-cache → `docs/superpowers/memory/churn-cache.json`.** One `git log --numstat`
   pass; exclude generated/vendor/merge/format commits; compute **normalized** per-path churn
   (commits/age, size-normalized) + fix/revert density; incremental via a stored last-SHA. O(1)
   read at pre-eval. Escalation-only: `hot→escalate`; low/absent/insufficient **never** "safe."
3. **Thin ADR-capture → `docs/superpowers/adr/NNNN-slug.md`.** A lightweight command captures a
   genuine decision-with-alternatives-and-consequences, drafts `context/decision/consequences`,
   **draft→human-confirm** (reference-hallucination risk documented), commits, and is auto-indexed
   by the existing FTS5 lane (already indexes `docs/superpowers/**`). C4/SAD explicitly out of
   scope.

## 6. Triage-outcomes stream (Tier 2 data foundation)

`docs/superpowers/memory/triage-outcomes.jsonl` — append-only, one line per pre-eval'd request:

```json
{"event":"predicted","ts":"…","pre_eval_id":"2026-07-11T181200Z-make-button-red-a1b2","taxonomy_sha":"…","difficulty_band":"low","impact_band":"low","decision":"FASTPATH_ELIGIBLE","localization":{"resolved_paths":["…"],"fan_out":1,"flags":[]}}
{"event":"bind","ts":"…","pre_eval_id":"2026-07-11T181200Z-make-button-red-a1b2","run_id":"2026-07-11-make-button-red"}
{"event":"actual","ts":"…","pre_eval_id":"2026-07-11T181200Z-make-button-red-a1b2","run_id":"2026-07-11-make-button-red","diff_files":1,"diff_lines":3,"sensitive_hit":false,"escalated":false,"review_result":"approved","test_result":"pass"}
```
(THREE separate appended lines joined on `pre_eval_id` — never one mutated object.)

Modeled as a **three-event protocol joined on `pre_eval_id`** (AC-3 — jsonl is append-only, and no
`run_id` exists at prediction time so the join key is the write-once `pre_eval_id`): `predicted` at
`PRE_EVAL_DONE`, `bind` (`pre_eval_id`→`run_id`) when the run dir is created, `actual` at
`MERGED`/`ESCALATION`. A declined/full-pipeline request has `predicted` and no `bind`. Duplicate/
out-of-order: last-writer-wins per `(pre_eval_id, event)`; a missing `actual` excludes that row from
precision (logged, never fabricated). Tier 2 joins on `pre_eval_id`, reads with `min_sample_count` +
a deterministic similarity rule (resolved-path category + request class). **Cohort-separated
counterfactual guard:** only **accepted fast-path** outcomes may support a healthy/lowering signal;
full-pipeline outcomes contribute **escalation evidence only** — at launch all entries are
full-pipeline, so Tier 2 stays escalation-only until fast-path outcomes accrue. Never a routing input
beyond this triage boundary; the stream is triage-only telemetry.

## Acceptance Criteria (feature-level; gate the final integration review)

1. The "make button X red" CSS case routes to `FULL_PIPELINE` via override #3 (shared-token/a11y),
   proven by a validator fixture — closing the prior mis-analysis.

   truth-table is fully testable with no live model call.
3. Pre-eval makes **zero** model calls on any request that hits a Layer-A override; T3 is reached
   only when `T1 unclassified ∧ T2 insufficient`; the whole stage respects a declared token-cap and
   aborts to `FULL_PIPELINE` on overrun.
4. `pre-eval.json` + `triage-outcomes.jsonl` are written **and committed** before state advances;
   `/v:status` renders the pre-eval decision; nothing is lost on worktree cleanup.
5. Tier 2 is escalation-only until `min_sample_count` *fast-path-taken* calibrated outcomes exist;
   legacy full-pipeline outcomes never lower ceremony (counterfactual guard test).
6. A fast-path run keeps: committed manifest, scope gate, the test floor, and a proportionate
   (1 combined SPEC+QUALITY pass, vacuous INTEGRATION) review. Sentinel audit skip-records validate.
7. Post-hoc: a fast-path diff that touches a sensitive path / exceeds size / adds control-flow
   transitions to `ESCALATION_REQUIRED`, preserves the patch, and rejoins the full pipeline —
   proven by a fixture.
8. `/v:onboard` drafts the taxonomy (incl. sensitive path-list) + the normalized churn cache behind
   the existing human GATE, never auto-applying; churn is normalized and escalation-only.
9. ADR-capture writes a human-confirmed `docs/superpowers/adr/*.md` that becomes FTS5-recallable; no
   C4/SAD generation exists.
10. Request-level score only ever OFFERS a choice (`ask`), never silently auto-routes; `off` is a
    hard kill-switch. No fabricated cost/token metric appears in any pre-eval output.
11. A remembered per-category choice (AC-11) skips the OFFER for that category only; every
    fail-closed override + post-hoc diff escalation still fires — proven by a fixture where a
    `css-only`-remembered request still escalates on a shared-token/a11y hit.
12. `/v:status` renders fast-path precision + escalation-rate from the `triage-outcomes` join
    (git-derived actuals), showing "insufficient samples" below the floor — never a fabricated
