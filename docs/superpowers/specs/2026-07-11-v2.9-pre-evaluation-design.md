# v2.9 ‚Äî Pre-Evaluation & Proportionate Fast-Path ‚Äî Design Spec

**Recon:** none (this design was grounded by a five-pass research corpus + an independent
cross-model design critique instead of a Trigger-0 recon; see References).

**Status:** design approved section-by-section (2026-07-11). Next: `writing-plans` ‚Üí
disjoint partition map ‚Üí manifest ‚Üí dispatch ‚Üí three-pass Review Gate ‚Üí Codex build rounds.

**Goal:** Add a fast, cheap **Pre-Evaluation** stage before Trigger 0 that scores each request
on two axes (difficulty, impact) from tiered deterministic evidence, and ‚Äî only when a change is
provably trivial and low-impact ‚Äî OFFERS the user a proportionate fast-path instead of the full
brainstorm ‚Üí 3 pre-flights ‚Üí partition ‚Üí 3-pass-review pipeline. Trivial fixes stop being a
Nolan odyssey; nothing risky ever silently skips ceremony.

**Architecture:** A new pre-brainstorm stage writing a separate machine-readable record
(`pre-eval.json`), a deterministic scoring truth-table (no raw LLM magnitude), a bounded
read-only localization step, three new state-machine states with a real escalation path, a
fast-path skip-contract that keeps the manifest + scope-gate + a proportionate review, and
onboarding extensions that manufacture the taxonomy + churn signals the scorer reads.

**Tech Stack:** Python stdlib (scoring engine, localization, churn), YAML (taxonomy), JSONL
(triage-outcomes, append-only), the existing manifest/state-machine/scope-gate substrate, the
existing FTS5 V-memory index (ADR indexing), one `light`-tier model call (Tier 3 only, rare).

---

## Global Constraints

- **Model policy:** Opus by default; `light`-tier T3 classify may run Sonnet; **never Haiku**
  (enforced by `lint-frontmatter.py` + CI). Reviewers are **always** `deep`/opus (manifest
  invariant #4) ‚Äî unchanged.
- **No fabricated metrics** (anti-ruflo): the displayed 1-10 is a band-midpoint label, never a
  token/cost estimate; no cost numbers anywhere in pre-eval output.
- **External-launch invariant:** any external CLI (T3 if routed to codex, churn is pure git) goes
  through `scripts/compound-v-run-with-timeout.py` with `stdin </dev/null`.
- **Git-derived enforcement, never model self-report:** the scope gate and the post-hoc diff
  re-classification read git, not any worker's claim. Unchanged.
- **Commit-discipline (v2.6.4):** `pre-eval.json`, `triage-outcomes.jsonl`, `churn-cache.json`,
  the taxonomy, and ADR docs are **written AND committed** ‚Äî an uncommitted artifact vanishes on
  `git worktree remove` and never indexes into V-memory.
- **Recon boundary preserved:** Pre-Evaluation is a **routing/triage** artifact; recon
  (`phase-0-recon.md`) stays evidence-only, five-section, never a routing input. They are separate
  records. Recon's contract is not modified.

## Iron Invariants (honesty constraints ‚Äî not scope-negotiable)

1. **No raw LLM magnitude.** The LLM only ever classifies into pre-declared bands; the score is
   assembled by deterministic logic. (Evidence: identical inputs scored 6.8/5.2/7.5 across runs.)
2. **Localization before any `low`.** A `low` verdict is impossible until a bounded read-only step
   resolves real paths/tokens/fan-out. "make button X red" may be a global design token or an a11y
   contrast state ‚Äî discovered before the decision, not after.
3. **Tier 2 self-gates on `min_sample_count` + counterfactual guard.** Historical outcomes may only
   lower ceremony once enough *fast-path-taken* outcomes have accrued and calibrated. Legacy
   full-pipeline successes are counterfactual and may only ESCALATE, never fast-path. At launch,
   with no data, Tier 2 is escalation-only by construction.
4. **Request-level score only OFFERS, never auto-routes.** (Evidence: text-only prediction AUC
   0.52-0.57 vs structural 0.957; every production triage system keeps a human-confirm step.)
5. **Fail-closed everywhere.** Any ambiguity, missing data, tier disagreement, token-budget
   overrun, or `unknown` axis ‚Üí `FULL_PIPELINE`. Pre-eval can only ever *save* work on the proven
   trivial path; it never fails open.
6. **The scope gate is never skippable, and the prediction is never trusted once a diff exists.**
   Post-hoc the real diff is re-classified; a violation escalates to the full pipeline.

## 1. Pipeline placement, records, and states

New stage, first in line:

```
request
  ‚îî‚îÄ‚ñ∫ PRE-EVAL (new) ‚îÄ‚îÄ‚ñ∫ Trigger 0 recon ‚îÄ‚îÄ‚ñ∫ brainstorming ‚îÄ‚îÄ‚ñ∫ Trigger 1 ‚îÄ‚îÄ‚ñ∫ partition ‚îÄ‚îÄ‚ñ∫ ...
        ‚îú‚îÄ bounded read-only LOCALIZATION (resolve real files/tokens/fan-out)
        ‚îú‚îÄ Tier 1 taxonomy ¬∑ Tier 2 history (min_sample-gated) ¬∑ Tier 3 light LLM-classify (rare)
        ‚îú‚îÄ churn signal (escalation-only) from onboarding cache
        ‚îî‚îÄ writes SEPARATE machine-readable pre-eval.json (+ triage-outcomes.jsonl)
```

**Records:**
- `docs/superpowers/execution/<run-id>/pre-eval.json` ‚Äî control-plane state (see ¬ß2 for shape).
- `docs/superpowers/memory/triage-outcomes.jsonl` ‚Äî append-only task-level learning stream (¬ß4).
- Recon's own doc + `recon-outcomes.jsonl` are untouched.

**New state-machine states** (extend the linear `SPEC_READY‚Üí‚Ä¶‚ÜíMERGED` + `BLOCKED`):

| State | Reached when |
|---|---|
| `PRE_EVAL_DONE` | Score computed, `pre-eval.json` written **and committed**. Precedes `SPEC_READY`. |
| `FASTPATH_DISPATCHED` | Fast-path eligible, user accepted; the single-job fast-path manifest dispatched. |
| `ESCALATION_REQUIRED` | Post-hoc diff re-classification failed; patch preserved as evidence, pipeline rejoins the full path. |

Transitions:

```
PRE_EVAL_DONE ‚îÄ(eligible ‚àß accept)‚îÄ‚ñ∫ FASTPATH_DISPATCHED ‚îÄ‚ñ∫ [scope-gate + parse/test]
    ‚îú‚îÄ(diff clean)‚îÄ‚îÄ‚îÄ‚îÄ‚ñ∫ REVIEWED (1 combined pass) ‚îÄ‚ñ∫ MERGED
    ‚îî‚îÄ(diff violates)‚îÄ‚ñ∫ ESCALATION_REQUIRED ‚îÄ‚ñ∫ PREFLIGHT_DONE ‚îÄ‚ñ∫ PARTITION_VERIFIED ‚îÄ‚ñ∫ ‚Ä¶ (normal)
PRE_EVAL_DONE ‚îÄ(not eligible ‚à® decline ‚à® off)‚îÄ‚ñ∫ SPEC_READY ‚îÄ‚ñ∫ ‚Ä¶ (normal)
```

`state.json` is written after every transition; `pre-eval.json` is committed before advancing
(same commit-discipline as recon ‚Äî else worktree cleanup deletes it and `/v:status` goes blind).

## 2. Scoring truth-table (deterministic; no raw LLM magnitude)

Two axes computed **separately**, each ‚Üí band `{low, medium, high, unknown}`. Within an axis:
**conservative-max across tiers, never average down** ‚Äî a single strong high signal must not be
diluted. "Coefficients" survive as per-tier authority weighting; "multi-dimensional" survives as
the two axes never merging into one number.

### Layer A ‚Äî hard overrides (fail-closed, ordered; first match ‚Üí `FULL_PIPELINE`, zero further cost)

| # | Condition | Rationale |
|---|---|---|
| 1 | localization = `failed` ‚à® `ambiguous` | paths unknown ‚Üí cannot judge |
| 2 | any resolved path matches the **sensitive path-list** (taxonomy) | auth/payments/PII/a11y/migrations/infra |
| 3 | `is_shared_token` ‚à® `is_generated` ‚à® `is_a11y_state` | "button" = global token / contrast |
| 4 | semantic-vs-path disagreement (T1 `low` but T3 `user-facing-major`, or converse) | tier disagreement escalates |
| 5 | `churn.hot` on any resolved path | churn ESCALATES only; low/insufficient never lowers |
| 6 | any axis = `unknown` | no signal ‚Üí full pipeline |

Most non-trivial requests hit an override with **zero LLM calls**.

### Layer B ‚Äî positive fast-path gate (only if no override fired)

`FASTPATH_ELIGIBLE` ‚ü∫ `difficulty == low` ‚àß `impact == low` ‚àß `fan_out ‚â§ threshold` ‚àß
**exactly one literal normalized path** (manifest single-path partition rule ‚Äî no glob metachar,
not shared/generated/config/migration).

### Band composition (conservative-max)

**Difficulty:** T1 taxonomy is primary. T2 (`matched ‚àß n‚â•min_sample ‚àß healthy`) *corroborates* low
but never independently creates it pre-calibration; `unhealthy` raises. T1-unclassified ‚àß
T2-insufficient ‚Üí T3 classify (`plumbing‚Üílow`, `user-facing-minor‚Üílow/med`,
`user-facing-major‚Üíhigh`, `unknown‚Üíunknown`). Tier conflict ‚Üí higher band.

**Impact** (weaker text proxies ‚Üí asymmetric): T1 taxonomy primary and most authoritative; T3
impact is low-confidence and may only **raise**, never lower below T1.

### Derived 1-10 (display only)

`low‚Üí1-3, med‚Üí4-6, high‚Üí7-10`; show band-midpoint (or per-tier max) **per axis**. A label computed
*after* the decision ‚Äî never the gate.

### Declared missing-data behavior

| Missing | Behavior |
|---|---|
| taxonomy file | T1 unclassified all paths ‚Üí degrade to T2/T3 |
| churn cache | churn signal absent (never escalates, never lowers ‚Äî safe) |
| triage-outcomes empty / n<min_sample | T2 insufficient (escalation-only) |
| localization fails | override #1 ‚Üí FULL_PIPELINE |
| token-budget overrun | abort pre-eval ‚Üí FULL_PIPELINE |

### `pre-eval.json` shape (records everything needed to reproduce + learn)

`{run_id, request_slug, taxonomy_version, ts, difficulty:{band, display}, impact:{band, display},
tiers_signalled:[‚Ä¶], localization:{resolved_paths, fan_out, flags, confidence},
override_fired:<id|null>, decision:<FASTPATH_ELIGIBLE|FULL_PIPELINE>, min_sample_status}`.
`confidence` = signalled tiers + localization_confidence, shown as **evidence**, never a gating
number.

## 3. Cost/speed budget (Pre-Eval must be near-free by construction)

Tier order guarantees common cases exit with **zero model calls**: localization (ripgrep + glob +
git), T1 (YAML lookup), churn (cache lookup), T2 (local FTS5 query), Layer-A override ‚Äî all
deterministic. **T3 is the only model call, reached only when `T1 unclassified ‚àß T2 insufficient`.**

Three budget rules:
1. **Localization is bounded** ‚Äî not mini-archaeology. Resolve symbol/selector/token via
   ripgrep+glob, hard cap on files inspected + timeout. Cannot resolve within cap ‚Üí `ambiguous` ‚Üí
   override #1 ‚Üí full pipeline (degrades to "just do it normally," never spends more).
2. **T3 = `light` tier** (Sonnet, never Haiku), one classify call, tiny input (request text +
   resolved paths + taxonomy categories), bounded enum output, no thinking budget.
3. **Declared token-cap on the whole stage.** Overrun ‚Üí abort ‚Üí `FULL_PIPELINE` (fail-safe). Worst
   case: pre-eval costs its bounded budget, then the normal pipeline runs ‚Äî the same pipeline that
   would have run anyway.

## 4. Fast-path skip-contract

**Skipped:** heavy brainstorm (‚Üí minimal confirm); 1A (localization already did archaeology-lite);
1B/1C via **validated sentinel skip-records**; multi-job partition (single literal path ‚Üí auto
single-job manifest).

**Never skipped:** manifest materialization + commit; git-diff scope gate; the test floor; a
**proportionate** review (not zero).

**Sentinel audits.** `audits:{archaeology,domain,library}` still required by the manifest; fast-path
writes a tiny skip-record per audit: `{skipped:true, reason:fastpath, localization:<path>,
taxonomy_version}`. Contract satisfied (paths exist, readable); skip is auditable, not a silent
null. `compound-v-validate-manifest.py` learns to accept a skip-record.

**Review-pass matrix.** Reviewer-invariant (reviewers ‚áí deep/opus) is untouched ‚Äî savings come from
skipping brainstorm + 3 pre-flights + partition, NOT from a cheaper reviewer. The review itself:
INTEGRATION pass is vacuous (single job, no seams) ‚Üí auto-pass with recorded rationale; SPEC +
QUALITY ‚Üí **one combined diff-scoped pass** (Opus, honest, single-pass on a tiny diff).

**Test floor** (from the no-tooling ladder research): tier-1 real tests if configured ‚Üí else tier-2
language parse-check on touched file (`py_compile`/`node --check`/`tsc --noEmit`/`go build`/`ruby
-c`/`php -l`) ‚Üí else tier-3 one cheap diff-read ‚Üí else full review. Scope-gate + this floor are the
non-skippable minimum.

**Manifest schema changes.** New optional `fast_path:{eligible, pre_eval:<path>,
localization:<path>, taxonomy_version}`; `audits` accept skip-records; single-job exception in the
validator ‚Äî **exactly one literal normalized path**, no glob metachar, not
shared/generated/config/migration (Codex's rule verbatim).

**Post-hoc escalation.** Once the diff exists, re-classify it with the strong diff-lane signal
(structural features; AUC 0.957 in the MSR-2026 evidence). Diff touches a sensitive path / exceeds a
size threshold / adds control-flow or signature changes ‚Üí `ESCALATION_REQUIRED`: preserve the patch
as evidence, run the skipped 1A/1B/1C + planning, then revalidate/reimplement ‚Üí merge. Real evidence
overrides a wrong prediction.

## 5. Onboarding extensions (present-then-confirm, human-gated, never auto-applied)

Into the existing 9-step `/v:onboard` (DIAGNOSE drafts ‚Üí GATE per-section diff ‚Üí WRITE/COMMIT/INDEX):

1. **Taxonomy-draft ‚Üí `.claude/compound-v-impact-taxonomy.yaml`.** Scan directory/module structure +
   stack; draft `path-pattern ‚Üí (difficulty_band, impact_band)` rows + a **sensitive path-list**
   section (auth/payments/PII/a11y/migrations/infra globs from the real repo structure ‚Äî the
   path-list that did not previously exist) + `taxonomy_version`. CSS rows carry the
   "shared-token/a11y caught by localization" caveat so override #3 fires rather than a naive `low`.
   Human edits/approves at GATE. Never auto-applied.
2. **Normalized churn-cache ‚Üí `docs/superpowers/memory/churn-cache.json`.** One `git log --numstat`
   pass; exclude generated/vendor/merge/format commits; compute **normalized** per-path churn
   (commits/age, size-normalized) + fix/revert density; incremental via a stored last-SHA. O(1)
   read at pre-eval. Escalation-only: `hot‚Üíescalate`; low/absent/insufficient **never** "safe."
3. **Thin ADR-capture ‚Üí `docs/superpowers/adr/NNNN-slug.md`.** A lightweight command captures a
   genuine decision-with-alternatives-and-consequences, drafts `context/decision/consequences`,
   **draft‚Üíhuman-confirm** (reference-hallucination risk documented), commits, and is auto-indexed
   by the existing FTS5 lane (already indexes `docs/superpowers/**`). C4/SAD explicitly out of
   scope.

## 6. Triage-outcomes stream (Tier 2 data foundation)

`docs/superpowers/memory/triage-outcomes.jsonl` ‚Äî append-only, one line per pre-eval'd request:

```json
{"ts":"‚Ä¶","run_id":"‚Ä¶","request_slug":"‚Ä¶","taxonomy_version":"‚Ä¶",
 "predicted":{"difficulty_band":"low","impact_band":"low","decision":"FASTPATH_ELIGIBLE"},
 "localization":{"resolved_paths":["‚Ä¶"],"fan_out":1,"flags":[]},
 "actual":{"diff_files":1,"diff_lines":3,"sensitive_hit":false,"escalated":false,
           "review_result":"approved","test_result":"pass"}}
```

`predicted` appended at `PRE_EVAL_DONE`; `actual` back-filled at `MERGED`/`ESCALATION` ‚Äî the loop
Tier 2 learns from. Tier 2 reads with `min_sample_count` + a deterministic similarity rule (match on
resolved-path category + request class). **Counterfactual guard in the data gate:** only entries
where fast-path was actually taken (or full-pipeline with recorded outcome) count as calibration; at
launch all entries are full-pipeline, so Tier 2 stays escalation-only until fast-path outcomes
accrue. Never a routing input beyond this triage boundary; the stream is triage-only telemetry.

## 7. Config (`.claude/compound-v.json`)

New keys (fail-closed defaults): `pre_eval.enabled` (default `true`), `pre_eval.fast_path`
(`ask` default | `off`), `pre_eval.min_sample_count` (default e.g. 5), `pre_eval.fan_out_threshold`,
`pre_eval.token_cap`. Malformed/unknown ‚Üí warn once, use defaults, never treat invalid as an
auto-route. The fast-path offer folds into the recon-research ask into **one** bounded interaction
when both fire (Codex: combine the questions).

## Acceptance Criteria (feature-level; gate the final integration review)

1. The "make button X red" CSS case routes to `FULL_PIPELINE` via override #3 (shared-token/a11y),
   proven by a validator fixture ‚Äî closing the prior mis-analysis.
2. Every Layer-A row and every band-composition rule has a deterministic validator fixture; the
   truth-table is fully testable with no live model call.
3. Pre-eval makes **zero** model calls on any request that hits a Layer-A override; T3 is reached
   only when `T1 unclassified ‚àß T2 insufficient`; the whole stage respects a declared token-cap and
   aborts to `FULL_PIPELINE` on overrun.
4. `pre-eval.json` + `triage-outcomes.jsonl` are written **and committed** before state advances;
   `/v:status` renders the pre-eval decision; nothing is lost on worktree cleanup.
5. Tier 2 is escalation-only until `min_sample_count` *fast-path-taken* calibrated outcomes exist;
   legacy full-pipeline outcomes never lower ceremony (counterfactual guard test).
6. A fast-path run keeps: committed manifest, scope gate, the test floor, and a proportionate
   (1 combined SPEC+QUALITY pass, vacuous INTEGRATION) review. Sentinel audit skip-records validate.
7. Post-hoc: a fast-path diff that touches a sensitive path / exceeds size / adds control-flow
   transitions to `ESCALATION_REQUIRED`, preserves the patch, and rejoins the full pipeline ‚Äî
   proven by a fixture.
8. `/v:onboard` drafts the taxonomy (incl. sensitive path-list) + the normalized churn cache behind
   the existing human GATE, never auto-applying; churn is normalized and escalation-only.
9. ADR-capture writes a human-confirmed `docs/superpowers/adr/*.md` that becomes FTS5-recallable; no
   C4/SAD generation exists.
10. Request-level score only ever OFFERS a choice (`ask`), never silently auto-routes; `off` is a
    hard kill-switch. No fabricated cost/token metric appears in any pre-eval output.
