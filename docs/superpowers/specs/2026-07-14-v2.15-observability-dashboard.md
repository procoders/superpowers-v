# v2.15.0 — Local observability dashboard (present-only static snapshot + ephemeral read-only server)

**Goal:** Close the plugin's biggest competitive gap (no observability UI) with a **present-only** dashboard over
`docs/superpowers/execution/**` — a static self-contained HTML snapshot for audit/sharing, plus an **opt-in,
ephemeral, read-only** localhost server for a live-updating web view — without a daemon, a persistent service, or
any write/control surface.

**Architecture:** One new pure-stdlib script `scripts/compound-v-dashboard.py` with two subcommands. `emit` reads the
run/epic JSON and prints ONE self-contained HTML file (data inlined, theme-aware, offline). `serve` runs an
ephemeral `http.server`-based read-only viewer bound to `127.0.0.1`, GET/HEAD-only, realpath-contained to the
execution root, that serves a live page which polls the JSON every few seconds. Read-only by design: **observe in
the UI, act via the CLI** — no merge/kill/edit control (that stays with the git-derived gates; it is the moat, not a
gap). Wired through `/v:status --html|--serve` + a thin `/v:dashboard` command.

**Tech Stack:** Python 3.9-safe stdlib only (`json`, `html`, `http.server`, `socketserver`, `argparse`, `pathlib`,
`urllib.parse`) — NO Flask/CDN/npm. Self-contained HTML: inline CSS/JS, no external fetch except (in `serve` mode)
same-origin JSON under the execution root.

## Global Constraints

- **Present-only / no daemon:** `emit` writes ONLY the HTML file (default `docs/superpowers/execution/dashboard.html`).
  `serve` is a FOREGROUND process the user launches and Ctrl-C's — never backgrounded, never auto-launched, never
  persisted, dies on SIGINT. Neither ever writes into a run dir, spawns an agent, or mutates state.
- **Read-only, no control surface:** the UI has NO action that changes anything — no merge/kill/retry/edit. `serve`
  handles **GET/HEAD only**; any other method → `405`.
- **Anti-ruflo (this is the identity — a dashboard that does not lie):** render ONLY what is in the state files.
  **No fabricated progress percentages** — only real counts (`N/M jobs done`). Usage is **measured-only**: a job with
  `usage.measured != true` (or absent) shows `—`, never a fabricated `0`. Timestamps come ONLY from the state files
  (`updated_at`, `started_at`, `last_progress_at`, `recorded_at`); never invent a wall-clock the data doesn't have.
- **Degrade-safe:** a run dir with only `manifest.yaml` (no `state.json` yet) renders honestly ("not dispatched /
  no state yet"). Malformed/partial JSON → an "unparseable" note on that card, never a crash. Empty execution root →
  honest "no runs yet".
- **Python 3.9-safe, stdlib only, LANG=C clean, `--selftest`.**

## Security (the `serve` mode — the one novel risk)

1. **Bind `127.0.0.1` ONLY** (never `0.0.0.0`, never a hostname) — unreachable from the network. Default port `8787`;
   if taken, auto-pick a free ephemeral port (bind to port 0, report the chosen URL). Print the exact `http://127.0.0.1:<port>/` to open.
2. **GET/HEAD only.** Subclass the handler; `do_POST`/`do_PUT`/`do_DELETE`/etc. → `405 Method Not Allowed`. No upload,
   no write path exists in the code at all.
3. **Realpath-contained to the execution root.** The served root is the execution dir (or a caller `--execution-root`).
   Resolve every requested path to an absolute realpath and REJECT (`403`) anything not strictly under the resolved
   root, anything reached via a symlink that escapes the root, and any `..` traversal. Serve ONLY files with a
   `.json` / `.html` / `.yaml` / `.yml` suffix (the dashboard + the state data); everything else → `404`.
4. **No directory-listing leak:** disable the default `http.server` directory index (or return a minimal, contained
   listing of only run dirs). Never expose the repo above the execution root.
5. **Ephemeral + clean:** `serve` runs until SIGINT (Ctrl-C), then shuts the socket down cleanly. It creates no files
   except (optionally) the dashboard HTML inside the execution root; it removes nothing it didn't create.

## Data sources (all degrade-safe, read-only)

- `docs/superpowers/execution/<run-id>/manifest.yaml` — job specs (id · backend · tier · effort · isolation ·
  depends_on · write_allowed).
- `<run-id>/state.json` — `phase` + per-job `status`/`isolation`/`worktree`/`session_id`/`failure_class` +
  circuit-breaker/cooldown fields (state-machine.md).
- `<run-id>/results/<id>.json` — `job_result` (status · blocked · files_changed · violations · summary · exit_code ·
  failure_class · optional `usage`).
- `<epic-id>/epic-state.json` — epic spine: `features[]` (id · status · run_id · attempts · disposition ·
  blocker_audit_due) · `autonomy{stance,watch,...}` · breakers (`total_attempts`/`no_progress_cycles`/`resume_count`) ·
  `blocker_ledger[]` (confirmed/SUSPECTED · category · families_agreeing) · `final_review`.

## What it renders

- **Index (top):** every run + epic under the execution root, each a card: id · feature/title · top status (phase or
  epic status) with a colored pill · counts (jobs/features done vs total) · the newest real timestamp. Sorted newest-first.
- **Per-run detail** (collapsible): the manifest job table — id · backend · tier · **status pill** · **scope-gate
  PASS/BLOCKED** (from `results[].blocked`/`violations`) · files_changed count · usage (measured-only) — plus the
  `depends_on` edges (a simple textual/CSS DAG, not a heavy graph lib).
- **Per-epic detail** (collapsible): feature table — id · status · run_id · attempts · disposition — plus the
  marathon/watch panel (breaker axes + resume_count + watcher-armed) and the **blocker ledger** (each entry:
  confirmed vs SUSPECTED, category, families_agreeing, evidence).
- **Live (serve only):** a small JS loop re-fetches the JSON every ~3 s and re-renders; a header shows "live · last
  updated <time>". `emit` (static) omits the poll loop and stamps "snapshot · generated from files as of <newest mtime>".

## Delivery / wiring

- `scripts/compound-v-dashboard.py emit [--execution-root DIR] [--out FILE]` — write the static snapshot; print its path.
- `scripts/compound-v-dashboard.py serve [--execution-root DIR] [--port N]` — ephemeral read-only live server; print the URL.
- `commands/v-status.md` gains `--html` (→ `emit`, prints the file to open) and `--serve` (→ `serve`, prints the URL);
  a thin `commands/v-dashboard.md` is the discoverable alias.
- `.gitignore`: ignore the generated `docs/superpowers/execution/dashboard.html` (a build artifact, not source).
- `README` one-liner + honesty note: "observe in the browser (read-only, localhost, ephemeral); control via the CLI".

## Acceptance criteria

- `emit` produces valid self-contained HTML (parses; no external `http(s)://` resource except inline data) rendering
  the runs/epics that exist under the root; a run with only `manifest.yaml` renders an honest "no state yet"; malformed
  JSON → an "unparseable" card, no crash; empty root → "no runs yet".
- **Anti-ruflo asserted:** no `%`-progress in output; a job with `usage.measured=false`/absent shows `—` not `0`; no
  timestamp appears that isn't sourced from a state file (selftest feeds fixtures with/without usage + timestamps and
  asserts the rendering).
- **`serve` security asserted (unit-level, no live network):** the request handler rejects a non-GET method (`405`),
  a `..`-traversal / absolute path / symlink-escape (`403`/`404`), and a non-`.json/.html/.yaml` suffix (`404`); the
  server object binds `127.0.0.1` (asserted on the constructed server address, not by opening a public socket).
- **Present-only asserted:** the script never writes outside the `--out` HTML (grep/AST: no write to a run dir, no
  `subprocess` that launches a browser/agent, no bind to `0.0.0.0`).
- `--selftest` green under LANG=C + Python 3.9; `2.15.0` lockstep (plugin.json + marketplace.json + CHANGELOG);
  frontmatter + manifest validators clean; scope gate clean.

## Anti-goals (YAGNI)

No control actions (merge/kill/retry) — CLI only. No websockets/SSE (simple poll). No graph/chart library (CSS DAG +
tables). No auth/multi-user (localhost-only, single dev). No persistence/history DB (the run dirs ARE the history).
No `0.0.0.0` / remote exposure. No auto-launch / no background.
