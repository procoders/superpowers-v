# v2.16.0 — Decision **memory + challenge** (falsifiable past-reasoning aid, NOT a "reason-as-me" clone)

**Goal (reframed after the Phase-1B domain audit).** Help the maker **recall their OWN dated past reasoning**
at a brainstorm fork — as *falsifiable history*, pulled on demand — and **always pair it with a divergent
challenge**, so surfacing "you decided X before" triggers *re-examination*, not confirmation. It is explicitly
**NOT** "let the brainstorm reason as Oleg" (the domain audit rated that framing high-hazard: it manufactures
preferences, builds an echo chamber, and opposes this project's own cross-model / anti-anchoring moat). The
user chose the **memory+challenge** framing; this spec builds only that.

**Why the reframe (domain audit, sourced).** Choice-blindness + RLHF-preference research show that *articulating
a reason can cement a preference that never existed*, and default/pre-select effects (49%→86%; GDPR Recital 32
names a pre-tick an invalid-consent dark pattern) mean even a "neutral" suggestion nudges. So v1 removes every
push/nudge/pre-select, captures the *why* unprompted, and makes recall a **doubt amplifier**, not an answer.

**Architecture.** One stdlib spine `scripts/compound-v-preferences.py` (`recall` / `capture` / `distill` /
`stats` / `purge` / `--selftest`) over a **LOCAL, personal** append-only `decisions.jsonl` — stored under
`~/.claude/compound-v/preferences/`, **never committed** (it is a dated behavioral profile of one person; git
distribution would egress it — the v2.6.2 precedent moved machine-local personal data OUT of the committed
config). Recall builds an **in-process FTS5 index over that local `decisions.jsonl`** (reusing V-memory's
`fts5_escape` crash-safe primitive), so it is immediate and decisions-scoped — NOT the shared git-tracked
prose index. Wired as a **pull** into the batched-elicitation surface; a `/v:preferences` command exposes
stats/distill/purge.

**Tech Stack.** Python 3.9-safe stdlib only (`sqlite3` FTS5, `json`, `datetime`, `argparse`); import
`compound-v-memory.py` (`fts5_escape`, `redact`, `SECRET_RE`/`PEM_RE`) and `compound-v-update-memory.py`
(`append_line`) by path — do not fork either. No new dependency, no service.

## Global Constraints

- **Memory, not authority. A MARK is allowed; a pre-TICK is not.** The dark pattern the audit forbids is a
  **pre-selected default** — an option already chosen that the human must actively un-choose (the 49%→86%
  default effect, GDPR-Recital-32 invalid consent). v1 NEVER does that: no option is ever pre-selected, and the
  human always chooses from a neutral state. What v1 DOES allow (opt-in `marked` mode) is a **falsifiable, dated
  MARK** on the option matching the past pattern — a *label* ("↩ your past pick: 4/5 · 2026-07-14"), never a
  recommendation, never a default. The distinction is load-bearing: a mark is information beside a neutral
  choice; a pre-tick is an answer you must override. The brainstorming HARD-GATE is untouched.
- **Three modes; default is the least-intrusive.** Config `brainstorm.preferences`:
  - **`off`** — nothing.
  - **`on-demand`** (default) — the human/driver PULLS history ("have I decided something like this before?"); no
    unsolicited surface. A pull can't nudge.
  - **`marked`** — at a *qualifying* fork the past-matching option carries the falsifiable dated badge, **always
    rendered together with the mandatory divergent challenge** (below), never pre-selected. This is the "show me
    what I did before, marked" affordance — with the badge and the challenge inseparable, so the mark triggers
    re-examination, not autopilot.
  In every mode the surface is suppressed on recon/high-novelty forks and demoted on drift (below).
- **The "why" is captured UNPROMPTED and never fabricated.** On capture, the human writes their rationale in
  free text FIRST (or skips → `why:null`). Candidate rationales may be offered ONLY afterward, stored as a
  distinct **`borrowed`** class, and are **excluded** from the "your reasoning" display and from the dominant-why
  distillation. A system-authored reason with a rubber-stamp is the exact ruflo the charter forbids.
- **Recall is a doubt amplifier (the moat-reinforcing invariant).** EVERY surfaced past decision is emitted
  **with a mandatory divergent counter-move**: an explicit "…but this time may differ because …" / an option you
  did not take last time / a reason the past choice could be wrong. Recall that cannot produce a challenge is
  suppressed. This is what turns "more Oleg" into "challenge Oleg."
- **Never fires where recon widens.** On a **recon-touched or high-novelty fork** (gate on the FTS5 similarity
  score — a low top-match score = novel), preferences are **suppressed entirely**. Trigger-0 recon (widen) and
  preference recall (narrow) never fire on the same fork.
- **Anti-ruflo:** counts only (`4/5 similar forks`), **never a fabricated confidence %**. Drift/staleness are
  surfaced, never hidden. Secret-scan (reuse `SECRET_RE`/`PEM_RE`) + a light PII check refuse a decision record
  or `why` carrying a credential/PII-shaped token.
- **Local + personal + purgeable.** Data lives under `~/.claude/compound-v/preferences/`, never in the repo,
  never shipped. Each pattern carries `first_seen` / `last_confirmed` / `expires_at`; `/v:preferences purge`
  wipes it in one command; stale patterns are banner-flagged, not silently trusted.
- **Python 3.9-safe, stdlib only, LANG=C clean** (`open(..., encoding="utf-8")` everywhere — the `append_line`
  lesson), tz-aware timestamps `datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%SZ")` (**never**
  `utcnow()`), no `match`/`case`, no `X|Y` unions. `--selftest` (auto-run by the v2.14.1 all-selftest CI gate).

## Data model (LOCAL: `~/.claude/compound-v/preferences/`)

`decisions.jsonl` — append-only (via `append_line`), one record per captured fork:

```json
{
  "id": "2026-07-15-a1b2",
  "captured_at": "2026-07-15T10:00:00Z",
  "question": "headless shim default posture?",
  "context_tags": ["safety", "default-vs-power"],
  "options": ["safe default + opt-in", "run full pipeline"],
  "chosen": "safe default + opt-in",
  "why": "safer default; user owns the risk",   // human FREE-TEXT, or null. Never inferred.
  "why_class": "unprompted | borrowed | none",   // 'borrowed' = tapped a candidate; weighted down, hidden from "your reasoning"
  "recall_shown": true,                           // was past history surfaced at this fork?
  "challenged": true,                             // was a divergent counter-move shown alongside?
  "changed_after_recall": false,                  // did the human choose DIFFERENTLY than their own past pattern? (the clean drift signal)
  "suppressed_reason": null                        // "recon-touched" | "high-novelty" | null
}
```

`preferences.md` — distilled, dated, **local** (regenerated by `distill`; NOT V-memory-indexed — it's personal):
per-pattern dominant choice, the **unprompted** rationales verbatim, sample size, `first_seen`/`last_confirmed`/
`expires_at`, and the recency-weighted disagreement rate with a stale/demoted banner where applicable.

## Behavior

- **`recall(question, options, context, mode)`** (PULL). Build the in-process FTS5 index over `decisions.jsonl`,
  `fts5_escape` the query, BM25-match. Returns `{shown, evidence:[dated past decisions + unprompted whys],
  challenge:[divergent counter-moves], marked_option, sample_n, disagreement_rate, suppressed_reason}`.
  `marked_option` (the option string that matches the dominant past pattern, with its `count`/`date`) is
  populated **only** when `mode == "marked"` and the fork qualifies — it is a **LABEL to badge, never a
  selection or default** (`on-demand` returns it null even when history exists — pull surfaces the evidence, not
  a mark). **`shown:false`** (with a `suppressed_reason`) when: the top similarity is below the novelty floor
  (high-novelty), the fork is recon-touched, no challenge could be produced, or the pattern is expired. There is
  **no field that marks an option as chosen/default** in any mode.
- **Present** (driver, in the elicitation surface): if `shown`, render the dated history as *falsifiable "here
  is what you did on N similar forks, dated"* — never as identity ("you always…") — **immediately followed by
  the divergent challenge**. In `marked` mode the `marked_option` additionally gets the dated badge (`↩ your
  past pick: N/M · date`) beside that option, but **nothing is pre-selected** — the human still picks every
  option from a neutral state on the normal surface ladder, and the badge and challenge always render together.
- **`capture(...)`** (after the human answers, at reconciliation): prompt for the **unprompted** free-text why
  first (skippable); only then may offer candidates (stored `borrowed`). Append the record (with `recall_shown`,
  `challenged`, `changed_after_recall`, `suppressed_reason`) via `append_line`; secret/PII-scan first. Refresh
  `preferences.md` via `distill`.

## Drift / anti-echo (measured honestly, in ALL modes)

- **Recency-weighted last-K disagreement**, not an all-time override ratio: `disagreement_rate` = fraction of the
  last K similar forks where `changed_after_recall == true` (or, when recall was suppressed as a holdout, where
  the clean choice differed from the stored pattern). A rising rate demotes the pattern (stops surfacing) and
  banner-flags it "your reasoning here may have shifted."
- **Holdout probe (clean signal).** A deterministic fraction of qualifying forks are **held out** — recall is
  deliberately suppressed and the human's *un-nudged* choice recorded (`recall_shown:false`, tagged holdout).
  Comparing holdout choices to the stored pattern gives an uncontaminated drift/agreement estimate the
  surfaced-choices can't (a surfaced choice is a possibly-nudged, possibly-learned-helpless sample).
- **Auto-expiry.** A pattern not confirmed within a staleness window (`expires_at`) stops surfacing until
  refreshed by new decisions.

## Acceptance criteria

- `capture` → `recall` roundtrip works IMMEDIATELY via the in-process FTS5 over the local `decisions.jsonl`
  (no reindex, no shared-corpus dependency); `fts5_escape` routes every user string (a question with `.`/`-`/`OR`
  does not throw).
- **No pre-tick / no default in ANY mode:** `recall` never marks an option as chosen/selected/default. `off`
  disables entirely; `on-demand` surfaces evidence only on a pull (`marked_option` null); `marked` populates
  `marked_option` as a **label** (a badge string with count+date) that the renderer places beside a *neutral*
  option — a selftest asserts no option is pre-selected and the badge is never presented as a recommendation.
- **Challenge-paired (both modes):** every `shown:true` recall — and specifically every `marked_option` — carries
  a non-empty `challenge`; a recall that cannot produce one returns `shown:false, suppressed_reason:"no-challenge"`
  (so a mark can never appear without its divergent counter-move).
- **Recon/novelty suppression:** a high-novelty fork (top FTS5 score below the floor) or a recon-touched fork
  returns `shown:false` with the matching `suppressed_reason`.
- **Unprompted why:** a `borrowed` (candidate-tapped) rationale is stored `why_class:"borrowed"`, weighted below
  `unprompted`, and excluded from the distilled "your reasoning" section; a skipped why stores `null` — never an
  inferred rationale. Output contains no `%` confidence, only counts.
- **Drift honesty:** injecting `changed_after_recall`/holdout disagreements past the window demotes the pattern
  and banners it in `stats`/`preferences.md`; the rate is recency-weighted last-K, not all-time.
- **Local + purge:** data resolves under `~/.claude/compound-v/preferences/` (never the repo); `purge` removes it;
  secret/PII scan refuses a credential-shaped record.
- All selftests green under LANG=C + Python 3.9; 2.16.0 lockstep; scope gate clean; no write outside partition.

## Anti-goals (YAGNI + the domain audit's red lines)

NO pre-tick / pre-selected default / default-nudge (a falsifiable dated MARK in `marked` mode IS allowed — it is
a label beside a neutral choice, always challenge-paired, never a selection). NO "reason as the creator"
auto-answering. NO inferred/borrowed-as-real
