# v2.16.0 — Decision-preference capture ("snapshot the maker's judgment model")

**Goal:** During the brainstorm/elicitation phase Compound V already intercepts, **accumulate the maker's
decision model** — capture not just *what* was chosen at a fork but the *why* — so future forks can reason
"the way the creator would": surface the past choice + rationale as **dated, falsifiable evidence** (and,
once a pattern is strong, a **pre-selected** option), while the human still decides. It is a new **evidence
lane** over the existing V-memory recall layer, governed by the same iron rule: **evidence for the human,
never an authority that auto-decides.**

**Architecture:** One deterministic stdlib spine `scripts/compound-v-preferences.py` (`recall` / `capture`
/ `distill` / `stats` / `--selftest`) over an append-only `decisions.jsonl` log; a distilled human-readable
`preferences.md` that **V-memory indexes** (it lives under `docs/superpowers/**`, so the CORE FTS5 lane picks
it up for free). Wired into the batched-elicitation point (`skills/compound-v/brainstorm-elicitation.md`):
`recall` before a fork, `capture` after the human answers. A `/v:preferences` command surfaces the model.

**Tech Stack:** Python 3.9-safe stdlib only (reusing the V-memory FTS5 engine for similarity matching); no
new dependencies, no service. Git-tracked storage under `docs/superpowers/preferences/`.

## Global Constraints

- **Evidence, never authority (the V-memory charter, extended).** A recalled preference INFORMS the fork; it
  **never auto-answers**. The brainstorming HARD-GATE (no implementation without an explicit human choice)
  is untouched — pre-select only *highlights* an option, the human still taps it.
- **Anti-ruflo — never fabricate the "why".** The system observes the CHOICE (real); the *rationale* lives
  in the human's head. Candidate rationales are presented as **hypotheses**; only the human's confirmed/edited
  answer (or an explicit "none") is stored. A guessed rationale is **never** persisted as fact. Counts only
  (`4/5 similar forks`), **never a fabricated confidence %/score**.
- **Anti-anchoring (aligned with Trigger-0, not against it).** Trigger-0 recon exists to *widen* a brainstorm
  (facts / ≥2-3 divergent directions, never a conclusion). A preference lane that says "you always pick X"
  would *narrow* — the opposite. So recall **widens**: it shows the past choice as ONE dated input beside the
  live options, and pre-select is gated + demotable (below) so a stale taste can never silently steer.
- **Opt-in, conservative default.** Config `brainstorm.preferences` = `off | evidence | preselect`, default
  **`evidence`** (show the past, never highlight) — like every CV feature, pre-select is opt-in.
- **Python 3.9-safe, stdlib only, LANG=C clean, git-derived, `--selftest`.** Storage is git-tracked prose/data;
  a secret scan gate on capture (a decision record must never persist a credential).

## The data model

`docs/superpowers/preferences/decisions.jsonl` — append-only, one JSON record per captured fork:

```json
{
  "id": "2026-07-15-a1b2",
  "captured_at": "2026-07-15T10:00:00Z",
  "question": "how should the headless shim's default posture work?",
  "context_tags": ["safety", "headless", "default-vs-power"],
  "options": ["safe default + opt-in", "run full pipeline", "deliberately-stalling"],
  "chosen": "safe default + opt-in",
  "why": "safer default; user owns the risk decision",   // human-CONFIRMED or null; never inferred-as-fact
  "why_source": "confirmed | edited | none",
  "preselect_shown": "safe default + opt-in",             // what recall highlighted, or null
  "was_override": false                                    // chosen != preselect_shown (the strongest signal)
}
```

`docs/superpowers/preferences/preferences.md` — the distilled, human-readable model (regenerated by
`distill`, **V-memory-indexed**): dated pattern clusters, each with its dominant choice, the confirmed
rationales verbatim, the sample size, and the current override-rate. This is what `/v:remember` recalls.

## Behavior (the flow, per fork)

1. **`recall(question, options, context)`** — match the live fork to past decisions via the **V-memory FTS5
   engine** (over question + option labels + context tags). Return
   `{pre_select | null, evidence:[dated past decisions], sample_n, override_rate, reason}`.
2. **Present.** In `preselect` mode with a qualifying pattern: show the fork with that option highlighted +
   a dated evidence note (`"prior 4/5 similar forks → X; you said: 'safer default' [2026-07-14]"`). In
   `evidence` mode (default) or a non-qualifying pattern: show the fork normally, with any past evidence
   appended, no highlight.
3. **Human answers** (match = confirm; different = **override**).
4. **Hypothesis→confirm for the "why".** Offer ≥2 candidate rationales (drawn from the pattern's prior
   confirmed whys and/or the option framings) + "свой / none". The human taps one, edits it, or skips. Only
   that answer (or `none`) is stored — never a fabricated rationale.
5. **`capture(...)`** — append the record (with `preselect_shown`, `was_override`, `why`, `why_source`);
   update the pattern's override-tracking. `distill` refreshes `preferences.md`; the next V-memory reindex
   picks it up.

## Pre-select gating + anti-echo-chamber (the core correctness)

- **Pre-select fires ONLY when** the matched pattern has **≥ N** similar past decisions (default `N=3`),
  they are **consistent** (same choice ≥ a majority threshold), **AND** the pattern's **override-rate is
  below a ceiling** (default `< 0.4`). Below N, inconsistent, or above the ceiling ⇒ **evidence-only** (or
  nothing). No fabricated confidence — the gate is on real counts.
- **Override-driven auto-demotion.** Every `capture` records whether the human took the highlighted option.
  Per-pattern `override_rate = overrides / preselects_shown`. Once it crosses the ceiling the pattern
  **stops pre-selecting** and `preferences.md` flags it: *"your taste may have shifted — N recent overrides."*
  Drift is surfaced, never hidden.
- **Anti-self-reinforcement.** ONLY human picks update the model. A pick that **matched** the pre-select is
  weighted **less** (a rubber-stamp-under-suggestion), while an **override is the strongest signal** (it is
  corrective) — so confirmations can't ossify a stale model and a genuine change of mind moves it fastest.
- **Dated + falsifiable.** Every evidence line carries its date; `stats` shows override-rates and demoted
  patterns so the human can audit and prune.

## Acceptance criteria

- `capture` → `recall` roundtrip: after 3 consistent similar decisions, `recall` returns that option as
  `pre_select` **only** in `preselect` mode; with 2 (below N) or in `evidence` mode it returns evidence-only
  (no pre_select).
- **Override-demotion:** injecting overrides past the ceiling flips the pattern to no-pre-select and marks it
  demoted in `stats`/`preferences.md`.
- **Anti-self-reinforcement:** a matched (rubber-stamp) pick weights less than an override in the distilled
  dominant-choice computation (asserted on a fixture where overrides should win).
- **Anti-ruflo:** a `capture` with no human-confirmed why stores `why:null` / `why_source:none` — never an
  inferred rationale; the rendered `preferences.md`/`recall` output contains **no `%`** confidence and only
  real counts. Secret-scan refuses a decision record containing a credential-shaped token.
- **Human-gate intact:** `recall` never returns an "answer"; pre_select is a highlight flag only; the
  brainstorm still requires the human's explicit choice.
- **V-memory:** `preferences.md` under `docs/superpowers/preferences/` is picked up by the CORE FTS5 lane
  (a `/v:remember` over a captured pattern surfaces it after a reindex).
- All selftests green under LANG=C + Python 3.9; 2.16.0 lockstep; scope gate clean; no write outside the
  partitioned sets.

## Anti-goals (YAGNI)

No auto-answering / no removing the human gate. No fabricated confidence scores (counts only). No inferring a
rationale the human didn't confirm. No cross-user model (this is the single maintainer's taste, explicitly).
No ML/embeddings-required path (the V-memory FTS5 CORE lane suffices; the optional dense lane, if bootstrapped,
just improves matching — degrade-safe). No routing influence (preferences never touch the deterministic
routing order — same firewall as V-memory).
