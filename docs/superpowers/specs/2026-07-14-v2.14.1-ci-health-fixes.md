# v2.14.1 — CI & health fix-batch (from the plugin health audit)

**Goal:** Close the two CI holes the health audit found — 25 of 29 script selftests never run in CI, and the
intra-plugin cross-ref guard is silently dead (piped-`while` subshell bug) — plus trivial dead-code and
doc-drift cleanup. No new feature; a patch release.

**Architecture:** Four disjoint workstreams. **W1** hardens `.github/workflows/validate.yml`. **W2** deletes
dead code + gitignores a stray file. **W3** adds `--selftest` to the two uncovered scripts. **W4** fixes
stale/misleading docs. All verified: every one of the 29 existing selftests PASSES locally under Python
**3.9.6** (the CI floor), and all 8 tracked run-manifests validate — so wiring them into CI cannot break it.

## Global Constraints
- Python 3.9-safe, stdlib only. Opus-default / NEVER Haiku. Anti-ruflo. Two-command commit discipline.
- CI changes must be verified locally (run the exact bash/selftest) before commit — a broken CI change is
  worse than the gap it closes.

## W1 — `.github/workflows/validate.yml` (the two CI holes)
1. **Run EVERY script `--selftest`, not 4.** Replace the four hardcoded selftest steps with a loop that
   discovers every `scripts/*.py` containing `--selftest` and runs it under the Python 3.9 floor
   (fail the build on any non-zero). Auto-covers future scripts. (All 29 pass under 3.9.6 today.)
2. **Validate the real run-manifests + the rejection path.** Validate every tracked
   `docs/superpowers/execution/*/manifest.yaml` (8 today, all valid) in addition to the example; keep the
   example. (The validator's own `--selftest` — which exercises the *rejection* path — is now covered by #1.)
3. **Fix the dead cross-ref guard.** The inner `grep … | sed … | while read link; do … fail=1 … done` runs
   in a subshell, so `fail=1` never propagates and the guard never fails the build. Rewrite so the failure
   signal survives (collect dead links into a temp file / array, then test it), and **extend** the target
   match beyond `.md` to `.py`/`.sh`/`.json`/`.yaml` link targets. Depends on W4 fixing the 7 current dead
   links, else the now-working guard fails CI (correct behavior).
4. **Harden the vacuous `-f` guard.** The `if [ -f scripts/compound-v-validate-manifest.py ]` else-branch
   exits 0 with a warning — a rename would silently drop the gate. Make a missing required script a hard
   failure.

**AC (W1):** the workflow, run locally step-by-step, (a) runs all 29 selftests green under 3.9, (b) validates
all 8 manifests + example, (c) the cross-ref guard FAILS on a planted dead link and PASSES on a clean tree,
(d) `yaml.safe_load` parses the workflow.
