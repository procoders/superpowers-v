# MCP / external-tool recommender for `/v:onboard` (design, v2.5.1)

**Status:** approved for build (2026-07-01). The onboarding fast-follow from
[`2026-06-30-v-onboard-design.md`](2026-06-30-v-onboard-design.md) §10, now scoped as its own
patch. `.claude/rules` and bulk skill generation stay out (→ 2.5.2 / deliberately avoided).

## 1. What it is
A sub-capability of `/v:onboard`: from **signals in the repo** it recommends the right external
tool — biased toward an **already-authenticated CLI over an MCP server** when a solid one exists —
then writes `.mcp.json` through a **diff + human confirmation** with read-only / least-privilege
flags pre-filled, and emits a **loud, warn-only** lethal-trifecta note. Present-only during the
onboarding walk; the `.mcp.json` write is a separate, gated action. Nothing is auto-applied.

## 2. Why
Onboarding already studies the stack; the natural next step is "…and here are the tools that make
Claude effective on *this* repo, wired safely." The value is **least-privilege by default** (the
recommender pre-fills read-only/restricted flags, defusing the 2025 Supabase service-role toxic
flow at the source) and **CLI-over-MCP** where a CLI avoids a broad-token risk entirely (GitHub).

## 3. The signal → tool table (the core — deterministic, curated)
A **static table** in `scripts/compound-v-onboard.py` (curated like the model maps — no model
guesswork on tool names/flags). Each row: a repo signal → a recommendation. **Currency
(server names, packages, flags, CVE pins) is verified at build time** (Context7 / WebSearch), the
same doc-validation ethos as the rest of the plugin — MCP servers move fast.

| Signal (evidence) | Recommendation | Least-privilege / notes |
|---|---|---|
| `github.com` git remote | **`gh` CLI** (NOT a GitHub MCP) | avoids the broad-PAT toxic-flow entirely; surfaced as a CLI instruction, not a `.mcp.json` entry |
| Postgres DSN / `prisma` + `pg` | Postgres MCP | `--access-mode=restricted` |
| `@supabase/*` dep | Supabase MCP | `--read-only --project-ref <dev/branch DB>` |
| `playwright.config.*` | Playwright MCP | pin `≥ 0.0.40` (CVE-2025-9611) |
| fast-moving libs (react/next/tailwind/prisma/…) | Context7 | docs currency |
| `@sentry/*` dep | Sentry MCP | — |

Rows resolve to the **maintained vendor server**, never an archived reference entry. Each emitted
recommendation cites the **`file:line`** of its triggering signal (same anti-hallucination rule as
the onboarding citation gate). An unknown stack yields an **empty** recommendation set — honest,
not invented.

## 4. Components

### 4.1 Signal detector — `compound-v-onboard.py recommend-mcp`
Deterministic, stdlib. Input: the repo pack-manifest (from the existing `pack` subcommand) +
the git remote. Output (JSON): a list of
`{signal, evidence, kind: "cli"|"mcp", tool, package?, flags[], trifecta_risk: bool}`.
Reuses the existing pack/citation machinery; adds no new dependency.

### 4.2 `.mcp.json` writer (diff + confirm)
From the **confirmed** `kind: "mcp"` recommendations, build a `.mcp.json` (or a diff against an
existing one) with the least-privilege flags pre-filled. Present the diff; write **only** on
explicit human confirmation. `kind: "cli"` recommendations (e.g. `gh`) are surfaced as setup
instructions, never as `.mcp.json` entries. Never silent, never auto-apply. Merges into an
existing `.mcp.json` additively (never clobbers unrelated servers).

### 4.3 Lethal-trifecta warner
For any recommendation (or existing `.mcp.json` server) that combines **private-data access +
untrusted-content exposure + external-write capability**, emit a **named** warning with the
**specific remedy** (single-repo session / token scoping / read-only mode). Warn-only, loud,
actionable — **no hard refusal**; the user decides. Read-only defaults (§3) already defuse most
cases at the source.

### 4.4 Onboarding wiring
`skills/compound-v/onboarding.md` + `commands/v-onboard.md`: add the recommender as a
**present-only** step in the onboarding walk, with the gated `.mcp.json` write behind the human
approval gate. Move the "`.mcp.json` / MCP recommender = fast-follow / out-of-scope" notes to
**in-scope**; `.claude/rules` + bulk skills remain noted as out.

## 5. Invariants (non-negotiable)
1. **CLI-over-MCP bias** — a GitHub remote yields `gh` CLI, never a GitHub MCP server.
2. **Least-privilege pre-filled** — read-only / restricted / project-ref defaults on every MCP row.
3. **`.mcp.json` only via diff + confirm** — never auto-apply, never clobber unrelated servers.
4. **Lethal-trifecta is warn-only** — named warning + remedy; no hard refusal.
5. **Deterministic + evidence-cited** — the table is a static curated map; every recommendation
   cites its signal's `file:line`; an unknown stack yields an empty set (no invented tools).
6. **Currency verified at build** — server names / packages / flags / CVE pins are re-checked
   (Context7 / WebSearch) when this ships, and pinned in the table with the verification date.

## 6. Verification
- **`compound-v-onboard.py recommend-mcp --selftest`** with fixtures: a github remote → `gh` CLI
  (no MCP); a `@supabase/*` dep → Supabase MCP with `--read-only`; a Postgres DSN → Postgres MCP
  `--access-mode=restricted`; a private-data+untrusted+write combo → a trifecta warning; the
  `.mcp.json` diff writer produces valid least-privilege JSON and merges additively.
- **Dogfood on superpowers-v itself:** it has a `github.com` remote (→ `gh` CLI, **no** MCP),
  already uses Context7 (→ confirm, not duplicate), and has no Postgres/Supabase — verify the
  recommendation set matches reality with **no false MCPs** (the negative path).
- **Codex cross-model verification** of the signal→tool table + the lethal-trifecta logic (the
  model that writes ≠ the model that checks).
- Full regression (existing selftests + lint + CI version-lockstep) stays green.

## 7. Out of scope (2.5.1)
- Path-scoped `.claude/rules/*.md` writer (→ 2.5.2).
- Bulk skill generation (deliberately avoided — overlapping descriptions degrade auto-triggering).
- Any auto-apply of `.mcp.json` or CLI installs (the recommender presents; the human acts).
- Auto-authenticating any CLI/MCP or writing secrets.
